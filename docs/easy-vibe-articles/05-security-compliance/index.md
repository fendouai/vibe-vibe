# 安全、合规与局限性

> **定位**：理解智能体系统真实的风险面，设计生产级安全边界

---

## 两个基础问题

| 问题             | 回答                                     |
| ---------------- | ---------------------------------------- |
| **威胁从哪里来** | 提示注入、工具滥用、权限扩张、数据泄露   |
| **边界怎么设计** | 既保留能力，又把事故半径控制在可承受范围 |

---

## 文章索引

| 文章               | 核心观点                                   |
| ------------------ | ------------------------------------------ |
| AI代理威胁与缓解   | 风险是沿工具链、权限链不断扩散的具体攻击面 |
| 智能体架构安全边界 | 把能力范围、隔离方式、责任边界一起设计清楚 |

---

## 关键结论

> 不该只得到"要注意安全"这种正确但无用的结论，而应获得两种可落地的判断：
>
> 1. 识别最危险的接口与耦合点
> 2. 理解不同边界模型适合什么场景
