# AI 代理已经到来，威胁也随之而来

> **作者**：Jay Chen, Royce Lu (Palo Alto Networks Unit 42)  
> **来源**：https://unit42.paloaltonetworks.com/agentic-ai-threats/  
> **核心观点**：风险沿工具链、权限链不断扩散的具体攻击面

---

## 主要发现

| 发现                           | 说明                             |
| ------------------------------ | -------------------------------- |
| **攻击者未必需要显式提示注入** | 提示范围界定不当或指令缺乏约束   |
| **提示注入仍是最强攻击向量**   | 诱导泄露数据、滥用工具、改变行为 |
| **错误配置工具放大攻击面**     | 清理工具输入、严格访问控制       |
| **不安全代码解释器**           | 暴露任意代码执行和未授权访问     |
| **凭证泄露**                   | 服务令牌暴露引发身份冒用         |
| **不存在单独缓解措施**         | 必须采用分层纵深防御             |

---

## 九大攻击场景

| 场景                 | 威胁类型       | 缓解措施               |
| -------------------- | -------------- | ---------------------- |
| 识别参与代理         | 提示注入       | 提示强化、内容过滤     |
| 提取代理指令         | 提示注入       | 提示强化、内容过滤     |
| 提取工具模式         | 提示注入       | 提示强化、内容过滤     |
| 未经授权访问内部网络 | SSRF、工具滥用 | 工具输入清理、网络隔离 |
| 挂载卷数据泄露       | RCE            | 代码执行器沙箱         |
| 元数据服务令牌泄露   | 身份欺骗       | 沙箱、元数据服务限制   |
| SQL 注入             | 工具滥用       | 输入清理、参数化查询   |
| BOLA 攻击            | 访问控制破坏   | 严格授权检查           |
| 间接提示注入         | 提示注入       | 内容过滤、输出清理     |

---

## 防御策略

| 层面             | 措施                                   |
| ---------------- | -------------------------------------- |
| **提示强化**     | 禁止提取系统提示、响应超出范围请求     |
| **内容过滤**     | 检测提示注入、过滤敏感信息输出         |
| **工具安全**     | 输入验证、最小权限、网络隔离、安全扫描 |
| **代码执行沙箱** | 限制网络、过滤系统调用、最小权限容器   |
| **数据保护**     | DLP 方案、审计日志、机密管理、脱敏处理 |
| **纵深防御**     | 代理层+工具层+运行时层+监控层          |

---

## 关键结论

> **风险与框架无强绑定，多数漏洞来自设计方式和配置问题**

> **多层防御不可省略，单一缓解措施无法覆盖复杂攻击链**
