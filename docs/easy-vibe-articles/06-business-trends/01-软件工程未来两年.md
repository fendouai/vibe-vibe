# 软件工程的未来两年

> **作者**：Addy Osmani  
> **来源**：https://addyosmani.com/blog/next-two-years/  
> **核心观点**：角色分工、团队结构、交付方式整体重排

---

## 五个关键问题

| 问题                 | 核心要点                             |
| -------------------- | ------------------------------------ |
| **初级开发者问题**   | 入门级任务自动化，初级岗位收缩或反弹 |
| **技能问题**         | 核心编程能力退化或变得更重要         |
| **角色问题**         | 开发者收缩成审校者或升级为编排者     |
| **专家 vs 通才问题** | 窄专家被边缘化，T 型工程师占优       |
| **教育问题**         | 学位是黄金标准或灵活路径后来居上     |

---

## 初级开发者问题

| 数据                 | 说明                               |
| -------------------- | ---------------------------------- |
| 初级就业率下降       | 公司采用 AI 后六个季度内下降 9-10% |
| 大科技公司应届生招聘 | 过去三年砍掉约一半                 |
| 软件岗位预测增长     | 2024-2034 年增长约 15%             |

**两种未来**：

- 收缩：AI 替代入门级任务
- 反弹：软件渗透更多行业

---

## 技能问题

| 转变                       | 说明                               |
| -------------------------- | ---------------------------------- |
| 从"会不会自己把算法敲出来" | 到"能不能给 AI 清晰要求并判断输出" |
| 从"多打字"                 | 到"架构设计、复杂集成、边界情况"   |

> **最好的软件工程师，是知道什么时候不能轻信 AI 的人**

---

## 角色问题

| 悲观场景                       | 乐观场景                             |
| ------------------------------ | ------------------------------------ |
| 审校者：审查 AI 产出、看护合规 | 编排者：设计整体系统、编排 AI 和服务 |
| 创造者变成把关者               | 工程师升级成作曲家                   |

---

## T 型开发者

| 特征               | 说明             |
| ------------------ | ---------------- |
| **一两个深度专长** | 真正深挖的方向   |
| **跨域协作能力**   | 多学科团队粘合剂 |
| **端到端推进问题** | 减少交接成本     |

> **AI 放大通才优势：后端工程师可借助 AI 快速搭 UI**

---

## 教育问题

| 传统路径         | 灵活路径                     |
| ---------------- | ---------------------------- |
| 四年制 CS 学位   | 训练营、在线认证、作品集     |
| 课程落后行业需求 | 更聚焦现实技能               |
| 学位是默认凭证   | 45% 公司取消部分岗位学历要求 |

---

## 关键结论

> **真正不变的，只有变化本身。持续更新技能、拓宽能力边界、把注意力放在创造力、批判性思维、协作与判断上。**
