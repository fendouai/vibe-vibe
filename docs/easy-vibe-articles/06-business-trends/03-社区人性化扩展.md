# 用 AI 智能体扩展社区，同时保持人性化

> **作者**：Pauline P. Narvas  
> **来源**：https://vercel.com/blog/keeping-community-human-while-scaling-with-agents  
> **核心观点**：自动化后勤，保留人味

---

## Community Guardian 运营层

| 功能         | 说明                           |
| ------------ | ------------------------------ |
| **内容分析** | 分析新帖子，检查重复           |
| **任务分配** | 分配给最合适且有余量的团队成员 |
| **超时检查** | 48 小时无答复重新分配          |
| **状态跟踪** | 等待信息提醒、已解决自动更新   |

---

## c0 研究助手（智能层）

| 功能             | 说明                                      |
| ---------------- | ----------------------------------------- |
| **上下文收集**   | 检索知识库、文档、GitHub issues、历史讨论 |
| **情绪跟踪**     | 持续跟踪社区情绪和反复出现的障碍          |
| **产品反馈聚合** | 直接询问"当前最重要的产品反馈是什么"      |

---

## 23 天成果统计

| 指标           | 结果                                   |
| -------------- | -------------------------------------- |
| 帮助用户       | 281 位独立用户                         |
| 初始上下文收集 | 4,716 次响应                           |
| 话题复活       | 8 个"被遗忘"话题中 23 个收敛为解决方案 |
| 运营规模       | 最近两周运行超 1,400 次                |
| 重复检测       | 4 个重复话题，3 个自动关闭             |

## 技术栈

| 工具                 | 用途                               |
| -------------------- | ---------------------------------- |
| **AI Gateway**       | 调用 Claude，统一模型访问          |
| **Vercel Workflows** | 持久执行，每 10 分钟检查，间隙休眠 |
| **Chat SDK**         | 在 Slack 构建 c0 智能体            |
| **Claude**           | 主要 LLM 模型                      |

---

## 渐进式 Prompt 迭代

| Prompt | 进化                                               |
| ------ | -------------------------------------------------- |
| 初版   | "构建一个帮助处理社区日常运营的 agent，用 AI SDK"  |
| 细化   | "每 10 分钟触发一次，检查最新的话题"               |
| 进一步 | "确保我们每 4 小时轮换一次分配"                    |
| 特定   | "理解帖子内容，判断对方真正需要什么，再决定下一步" |

---

## 双层架构

| 层级       | 负责                    |
| ---------- | ----------------------- |
| **运营层** | Guardian 分配和跟踪帖子 |
| **智能层** | c0 研究与上下文补全     |
| **人工层** | 团队给出高质量回应      |

---

## 关键结论

> **自动化可以把重复劳动拿走，但它替代不了真诚的交流。少了分类、跟踪、提醒这些后勤负担之后，团队就能把时间放回更值得人来做的事情上：复杂排障、深度沟通、和社区成员建立真实关系。**
