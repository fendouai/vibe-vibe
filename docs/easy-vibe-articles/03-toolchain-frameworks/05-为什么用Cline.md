# 为什么我在 AI 工程中使用 Cline

> **作者**：Addy Osmani  
> **来源**：https://addyo.substack.com/p/why-i-use-cline-for-ai-engineering  
> **核心观点**：Cline 作为系统级工具，提供最大的灵活性和控制

---

## Cline 与其他工具的对比

| 工具               | 优势                                  | 限制                           |
| ------------------ | ------------------------------------- | ------------------------------ |
| **Cline**          | 模型灵活、系统集成、检查点、MCP可扩展 | Token定价需注意模型选择        |
| **Cursor**         | VS Code熟悉度、Composer多文件操作     | 较少系统级集成、有限运行时调试 |
| **WindSurf**       | 上下文感知好、UI精致、成本有效        | 功能较少、无浏览器自动化       |
| **GitHub Copilot** | 无限使用、VS Code无缝集成             | 基本多文件能力、无系统级操作   |

---

## Cline 的关键能力

| 能力               | 说明                                               |
| ------------------ | -------------------------------------------------- |
| **灵活上下文管理** | @file、@folder、@url、@problems 多种方式添加上下文 |
| **模型灵活切换**   | 支持 Anthropic、OpenAI、Gemini、DeepSeek、本地模型 |
| **检查点系统**     | 每次操作后自动捕获工作区状态，独立于 git           |
| **Computer Use**   | 启动浏览器交互、执行终端命令、实时响应系统输出     |
| **计划/执行模式**  | 先设计审查（计划），再直接实现（执行）             |
| **MCP 协议**       | 自定义工具扩展能力                                 |

---

## DeepSeek-R1 + Sonnet 混合工作流

| 模型                  | 用途                   | 成本优势                           |
| --------------------- | ---------------------- | ---------------------------------- |
| **DeepSeek-R1**       | 规划（$0.55/M tokens） | 擅长逻辑推理、架构分析             |
| **Claude 3.5 Sonnet** | 实现                   | 运行时调试、代码生成、浏览器自动化 |

> **这种组合可将成本降低高达 97%，同时保持甚至提高输出质量。**

---

## Cline 胜出的四个维度

| 维度             | 价值                               |
| ---------------- | ---------------------------------- |
| **模型灵活性**   | 策略性组合不同模型，成本可降 97%   |
| **系统集成**     | 与浏览器、终端、开发工具的深度集成 |
| **控制和可见性** | 人在回路中的批准和检查点           |
| **可扩展性**     | MCP 允许自定义工具和工作流程       |

---

## 适合场景

| 场景                    | 推荐                               |
| ----------------------- | ---------------------------------- |
| 复杂系统工作            | Cline——控制、灵活、系统级集成      |
| 以 GitHub为中心的工作流 | GitHub Copilot                     |
| 成本敏感的团队          | Cursor 或 WindSurf + Cline本地模型 |
| 最大灵活性和控制        | Cline                              |

---

## 关键结论

> **如果你正在构建复杂系统并希望 AI 辅助尊重工程原则，Cline 值得认真考虑。**
