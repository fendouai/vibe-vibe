# Claude Skills 完整构建指南

> **作者**：Anthropic  
> **来源**：https://docs.anthropic.com/en/docs/build-with-claude/skills  
> **核心观点**：Skill 是一组指令打包为文件夹，教会 Claude 如何处理特定任务

---

## Skill 的文件结构

```
your-skill-name/
├── SKILL.md          # 必需 - 主 skill 文件（YAML frontmatter + Markdown）
├── scripts/          # 可选 - 可执行代码
├── references/       # 可选 - 按需加载的文档
└── assets/           # 可选 - 模板、字体、图标
```

---

## 核心设计原则

| 原则           | 说明                                             |
| -------------- | ------------------------------------------------ |
| **渐进式披露** | 三级系统：frontmatter → SKILL.md 正文 → 链接文件 |
| **可组合性**   | 多个 Skills 可同时加载，不应假设独占             |
| **可移植性**   | Claude.ai、Claude Code、API 上工作方式相同       |

---

## YAML Frontmatter 必需字段

| 字段            | 要求                                             |
| --------------- | ------------------------------------------------ |
| **name**        | 仅 kebab-case，无空格或大写，与文件夹名匹配      |
| **description** | < 1024字符，包含"做什么"+"何时使用"，无 XML 标签 |

---

## Description 字段最佳写法

| ✅ 好的示例                                                                         | ❌ 不好的示例          |
| ----------------------------------------------------------------------------------- | ---------------------- |
| 分析 Figma 设计文件并生成开发者交接文档。当用户上传 .fig 文件或询问"设计规格"时使用 | 帮助处理项目           |
| 管理 Linear 项目工作流，包括冲刺规划。当用户提到"冲刺"、"Linear 任务"时使用         | 创建复杂的多页文档系统 |

---

## 三类用例

| 类别                     | 用途                 | 示例                     |
| ------------------------ | -------------------- | ------------------------ |
| **类别 1：文档创建**     | 生成一致高质量输出   | frontend-design skill    |
| **类别 2：工作流自动化** | 多步骤流程协调       | skill-creator skill      |
| **类别 3：MCP 增强**     | 工具访问的工作流指导 | sentry-code-review skill |

---

## 五种常见模式

| 模式                   | 使用时机                                             |
| ---------------------- | ---------------------------------------------------- |
| **顺序工作流编排**     | 用户需要按特定顺序执行多步骤流程                     |
| **多 MCP 协调**        | 工作流跨越多个服务（Figma → Drive → Linear → Slack） |
| **迭代改进**           | 输出质量通过迭代提升                                 |
| **上下文感知工具选择** | 相同结果根据上下文使用不同工具                       |
| **领域特定智能**       | Skill 添加超越工具访问的专业知识                     |

---

## 故障排除速查

| 问题         | 原因                     | 解决方案                        |
| ------------ | ------------------------ | ------------------------------- |
| 无法上传     | 文件名不完全是 SKILL.md  | 重命名为 SKILL.md（区分大小写） |
| 不触发       | description太通用        | 添加触发短语和具体任务          |
| 触发过于频繁 | description太宽泛        | 添加负面触发器，更具体          |
| MCP 连接失败 | API 密钥过期或工具名错误 | 检查认证状态和工具名称          |

---

## 关键结论

> **一旦你确认方案，把规范保存为 SPEC.md，并在后续需要时把相关片段重新喂给智能体。规范文件是跨会话存在的锚点。**
