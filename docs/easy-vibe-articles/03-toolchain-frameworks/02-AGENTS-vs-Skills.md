# AGENTS.md vs Skills：被动上下文为何胜过主动检索

> **作者**：Vercel Team  
> **来源**：https://vercel.com/blog/agents-md-outperforms-skills-in-our-agent-evals  
> **核心观点**：被动上下文胜过主动检索

---

## 评估结果

| 配置                   | 通过率   | 相比基线  |
| ---------------------- | -------- | --------- |
| 基线（无文档）         | 53%      | —         |
| Skill（默认行为）      | 53%      | +0pp      |
| Skill + 明确指令       | 79%      | +26pp     |
| **AGENTS.md 文档索引** | **100%** | **+47pp** |

---

## 为什么 Skills 没有被可靠触发

| 问题                 | 说明                         |
| -------------------- | ---------------------------- |
| **56% 案例从未调用** | 智能体可以访问文档但选择不用 |
| **零改进**           | Skill 存在但未使用           |

---

## 措辞脆弱性

| 指令                         | 行为                   | 结果           |
| ---------------------------- | ---------------------- | -------------- |
| "你必须调用 skill"           | 先读文档，锚定文档模式 | 错过项目上下文 |
| "先探索项目，然后调用 skill" | 先建立心智模型         | 更好的结果     |

---

## 为什么被动上下文胜出

| 因素             | 说明                             |
| ---------------- | -------------------------------- |
| **没有决策点**   | 不需要决定"我应该查这个吗？"     |
| **一致可用性**   | 每一轮都在系统提示中             |
| **没有排序问题** | 完全避免"先读文档 vs 先探索项目" |

---

## 上下文膨胀解决方案

| 方法             | 说明                         |
| ---------------- | ---------------------------- |
| **压缩索引**     | 40KB → 8KB（减少 80%）       |
| **管道分隔结构** | 目录路径映射到文档文件       |
| **按需读取**     | 索引指向文件，智能体按需读取 |

---

## 关键要点

| 要点                  | 说明                                                  |
| --------------------- | ----------------------------------------------------- |
| 被动上下文 > 主动检索 | 始终可用比按需调用更可靠                              |
| 压缩是关键            | 8KB 索引就足够达到 100% 通过率                        |
| 消除决策点            | 不让智能体决定"是否需要查文档"                        |
| 版本匹配很重要        | 确保文档与项目框架版本匹配                            |
| 两种方法互补          | AGENTS.md 适合横向框架知识，Skills 适合垂直特定工作流 |
