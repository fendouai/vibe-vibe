# AI 原型开发工具对比：v0、Bolt、Lovable

> **作者**：Addy Osmani  
> **来源**：https://addyo.substack.com/p/ai-driven-prototyping-v0-bolt-and  
> **核心观点**：三者各有定位，根据需求选择

---

## 工具定位

| 工具         | 定位             | 核心优势                               |
| ------------ | ---------------- | -------------------------------------- |
| **v0.dev**   | 快速 UI 原型开发 | Tailwind/MUI 集成顺畅、Vercel 部署无缝 |
| **Bolt.new** | 全栈浏览器 IDE   | 无需本地环境、实时调试、框架无关       |
| **Lovable**  | 引导式全栈体验   | Supabase 集成强、团队协作友好          |

---

## v0 的能力

| 功能          | 说明                                       |
| ------------- | ------------------------------------------ |
| **CLI集成**   | Blocks 功能提供 npx 命令直接安装组件到项目 |
| **Figma集成** | 导入设计文件，识别颜色和设计令牌           |
| **动画支持**  | 可包含 Framer Motion 等动画                |

---

## Bolt 的能力

| 功能         | 说明                                        |
| ------------ | ------------------------------------------- |
| **文件锁定** | 控制哪些文件可修改、哪些保持不变            |
| **请求中继** | 连接 CORS 会阻止的 API 和服务               |
| **本地版本** | Bolt.diy 支持自定义模型（Ollama、LMStudio） |

---

## Lovable 的能力

| 功能         | 说明                           |
| ------------ | ------------------------------ |
| **元素选择** | 从预览中直接选择元素并引用修改 |
| **知识库**   | 定义项目特定的细节和设计指南   |

---

## 共同挑战

| 挑战               | 影响                           |
| ------------------ | ------------------------------ |
| **依赖提示质量**   | 需求不清晰则生成偏离预期       |
| **成本和资源限制** | Token 使用限制影响大型项目迭代 |
| **最终需要"导出"** | 达到复杂性阈值后需转向本地编辑 |

---

## 用例推荐

| 用例             | 最佳选择                        |
| ---------------- | ------------------------------- |
| 快速前端组件原型 | v0                              |
| 快速全栈应用开发 | Bolt                            |
| 协作团队项目     | Lovable                         |
| 后端密集型应用   | Bolt 或 Lovable（Supabase集成） |

---

## 关键结论

> **这些工具已显著降低了从想法到可运行原型之间的摩擦。最终谁更适合你，取决于它在"自动化"和"开发者控制权"之间的平衡。**
