# 我们删掉了智能体 80% 的工具

> **作者**：Andrew Qu (Vercel)  
> **来源**：https://vercel.com/blog/we-removed-80-percent-of-our-agents-tools  
> **核心观点**：更多工具=更多决策负担

---

## 性能对比

| 指标            | 高级版（旧） | 文件系统（新） | 变化           |
| --------------- | ------------ | -------------- | -------------- |
| 平均执行时间    | 274.8s       | 77.4s          | **3.5 倍更快** |
| 成功率          | 4/5 (80%)    | 5/5 (100%)     | **+20%**       |
| 平均 token 使用 | ~102k        | ~61k           | **减少 37%**   |
| 平均步骤数      | ~12 步       | ~7 步          | **减少 42%**   |

---

## 问题根源

| 假设                       | 问题             |
| -------------------------- | ---------------- |
| 模型会在复杂 schema 中迷路 | 构建护栏约束推理 |
| 模型会做出错误的 join      | 预先过滤上下文   |
| 模型会幻想不存在的表名     | 添加验证逻辑     |

> **我们在替模型思考**

---

## 新技术栈

| 组件     | 说明                       |
| -------- | -------------------------- |
| **模型** | Claude Opus 4.5            |
| **执行** | Vercel Sandbox             |
| **工具** | 只一个：执行任意 bash 命令 |

---

## 经验教训

| 教训                     | 说明                             |
| ------------------------ | -------------------------------- |
| **别对抗重力**           | 文件系统是极其强大的抽象         |
| **我们在约束推理**       | Opus 4.5 这种约束反而成了负担    |
| **语义层本身就是好文档** | YAML 文件结构良好、命名一致      |
| **做减法是真实存在的**   | 最好的智能体可能是工具最少的那些 |

---

## 对智能体构建者的建议

| 建议                             | 说明                       |
| -------------------------------- | -------------------------- |
| **从最简单架构开始**             | 模型 + 文件系统 + 目标     |
| **只有在证明必要时才添加复杂性** | 不要预设问题               |
| **投资于文档和清晰命名**         | 这个基础比聪明的工具更重要 |
| **为六个月后的模型构建**         | 模型改进速度比工具更新更快 |

---

## 关键结论

> **每个工具都是你替模型做的一个选择。有时候模型会做出更好的选择。**
