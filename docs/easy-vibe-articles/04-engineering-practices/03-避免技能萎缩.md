# 在 AI 时代避免技能萎缩

> **作者**：Addy Osmani  
> **来源**：https://addyo.substack.com/p/avoiding-skill-atrophy-in-the-age  
> **核心观点**：将 AI 作为协作者而非拐杖，保持核心技能敏锐

---

## 技能萎缩的危险信号

| 信号             | 表现                                     |
| ---------------- | ---------------------------------------- |
| **调试绝望**     | 跳过调试器，直接向 AI 求助每个异常       |
| **盲目复制粘贴** | 粘贴无法自己实现或解释的代码             |
| **架构思维减弱** | 不愿意处理高层架构规划                   |
| **记忆减弱**     | 日常语法或概念逃离，因为自动补全总是填充 |

---

## 研究发现：批判性思维正在成为牺牲品

| 问题           | 研究（微软+CMU 2025）发现      |
| -------------- | ------------------------------ |
| 对 AI 高度信心 | 导致心理退居二线，"放开方向盘" |
| 长期依赖       | 独立解决问题能力减弱           |
| 解决方案多样性 | AI 倾向于提供同质化答案        |

---

## 过度依赖的长期风险

| 风险               | 影响                              |
| ------------------ | --------------------------------- |
| **批判性思维危机** | 工具不足时无法处理新问题          |
| **自我实现预言**   | 不加批判使用 AI，将自己降级为无关 |
| **团队动态影响**   | 指导和渗透学习受损                |

---

## 将 AI 作为协作者的七个策略

| 策略                | 做法                                      |
| ------------------- | ----------------------------------------- |
| **AI 卫生**         | 始终验证和理解，红队 AI 建议              |
| **基础知识不用 AI** | 设立"无 AI 日"，保持基本技能熵减          |
| **先自己尝试**      | 在询问 AI 前，先尝试解决问题 15-30分钟    |
| **增强代码审查**    | 对 AI 代码像审查同事代码一样              |
| **主动学习**        | 用 AI 解释不熟悉代码，尝试难倒 AI         |
| **学习日志**        | 跟踪经常向 AI 求助的事项，弥补知识差距    |
| **AI 配对编程**     | 你写主体，AI 提建议；或 AI 先起草，你重构 |

---

## 关键心态

| 心态                | 说明                                  |
| ------------------- | ------------------------------------- |
| **信任但验证**      | AI 是"过度自信且容易犯错"的初级开发者 |
| **人是回路把关人**  | 你是高级开发者，AI 是加速器           |
| **AI 放大你的能力** | 不是取代你的判断                      |

---

## 关键结论

> **明天最好的开发者将是那些没有让今天的 AI 让他们忘记如何思考的人。**
