# React 最佳实践：Vercel 十年优化经验

> **作者**：Shu Ding, Andrew Qu  
> **来源**：https://vercel.com/blog/introducing-react-best-practices  
> **核心观点**：优先级排序——先修复瀑布流和 Bundle，再优化微观性能

---

## 为什么需要这个框架

| 问题                  | 影响                      |
| --------------------- | ------------------------- |
| 异步操作变成串行执行  | 增加等待时间              |
| Bundle 体积随时间增长 | 每个页面多发送 JavaScript |
| 组件不必要的重复渲染  | 影响每个用户会话          |

## 八大性能类别

| 类别                | 影响评级 | 优先级 |
| ------------------- | -------- | ------ |
| **消除异步瀑布流**  | CRITICAL | 最高   |
| **Bundle 体积优化** | CRITICAL | 最高   |
| 服务端性能          | HIGH     | 高     |

## 关键优化示例

### 1. 避免不必要的异步等待

| 错误写法                       | 正确写法                       |
| ------------------------------ | ------------------------------ |
| 先等待数据，再判断分支是否需要 | 先判断分支，仅在需要时获取数据 |
| 两个分支都被阻塞               | 提前返回不阻塞不需要的数据     |

### 2. 并行化 await

| 场景                       | 优化                     |
| -------------------------- | ------------------------ |
| 串行等待无依赖的数据库调用 | 同时运行，总等待时间减半 |

### 3. 惰性状态初始化

| 问题                              | 解决方案                                                  |
| --------------------------------- | --------------------------------------------------------- |
| 每次渲染从 localStorage 解析 JSON | 使用 `useState(() => JSON.parse(...))` 仅初始化时解析一次 |

### 4. 合并循环迭代

| 问题                 | 解决方案       |
| -------------------- | -------------- |
| 同一消息列表扫描八次 | 合并为单次遍历 |
