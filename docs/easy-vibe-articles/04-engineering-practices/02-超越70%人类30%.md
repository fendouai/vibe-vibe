# 超越 70%：最大化人类 30% 的价值

> **作者**：Addy Osmani  
> **来源**：https://addyo.substack.com/p/beyond-the-70-maximizing-the-human  
> **核心观点**：人类剩下 30% 是决定成败的高杠杆判断

---

## 70% 问题

| AI 擅长    | AI 挣扎  |
| ---------- | -------- |
| 样板代码   | 边缘情况 |
| 常规函数   | 架构完善 |
| 大部分程度 | 可维护性 |
| 常见路径   | 异常输入 |

> **AI 处理的是"偶然复杂性"，人类处理的是"本质复杂性"**

---

## 持久工程技能

| 技能           | 说明                         |
| -------------- | ---------------------------- |
| **系统设计**   | 整体设计、组件交互、数据流   |
| **边缘情况**   | 批判性思维和远见             |
| **代码审查**   | 单元测试、集成测试、调试     |
| **调试能力**   | 重现、隔离原因、底层理解     |
| **上下文维护** | 系统思维、业务目标、遗留代码 |
| **沟通协作**   | 澄清需求、讨论权衡、协调工作 |
| **适应学习**   | 学习新工具、保持开放心态     |

---

## 高级开发者的优势

| 角色               | 说明                         |
| ------------------ | ---------------------------- |
| **架构师和总编辑** | 让 AI 处理第一稿，你完善输出 |
| **力量倍增器**     | AI 处理大量机械变化          |
| **指导设定标准**   | 教初级有效使用 AI            |
| **领域精通**       | 历史知识、业务需求           |
| **软技能和领导力** | 沟通、设计会议、判断         |

---

## 初级开发者的成长路径

| 建议                 | 说明                                 |
| -------------------- | ------------------------------------ |
| **学习基础知识**     | 不跳过"为什么"，追问 AI 为什么这样写 |
| **练习解决问题**     | 定期"无 AI 日"，自己调试 AI 生成代码 |
| **专注于测试和验证** | 为代码写测试，挑战 AI 输出           |
| **培养可维护性眼光** | 主动重构 AI 的代码                   |
| **发展提示技能**     | 学会向 AI 提问、给它适当上下文       |
| **寻求反馈和指导**   | 问高级开发者为什么偏好某个方案       |

---

## 关键结论

> **AI 是编程力量倍增器，但被倍增的是你的专业知识。保持专业知识的敏锐性，并将其置于开发过程的中心。**
