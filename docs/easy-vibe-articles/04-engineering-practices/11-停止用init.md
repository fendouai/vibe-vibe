# 停止为 AGENTS.md 使用 /init

> **作者**：Addy Osmani  
> **来源**：https://addyosmani.com/blog/agents-md/  
> **核心观点**：AGENTS.md 应是"代码库里尚未产品化的约束清单"，而非永久配置

---

## 两项研究发现

| 研究                       | 发现                                                    |
| -------------------------- | ------------------------------------------------------- |
| **Lulla 等人（2026）**     | AGENTS.md 使运行时间减28.64%，token减16.58%（人工编写） |
| **苏黎世联邦理工（2026）** | LLM 生成的文件使成功率降2-3%，成本增20%以上             |

---

## 研究矛盾的解释

| 类型                     | 效果                                   |
| ------------------------ | -------------------------------------- |
| **人工编写的 AGENTS.md** | 提供真正项目特定知识，节省智能体时间   |
| **LLM生成的 AGENTS.md**  | 与智能体可自行发现的信息冗余，增加噪音 |

---

## 粉红大象问题

| 问题                 | 影响                                                           |
| -------------------- | -------------------------------------------------------------- |
| **锚定效应**         | 提到 tRPC 即使只在少数遗留端点，模型在每次提示都有 tRPC 上下文 |
| **上下文稀释注意力** | 添加的每一样东西都与实际任务竞争                               |

---

## 什么真正值得占一行

| ✅ 值得写                        | ❌ 不值得写（AI能自己发现）        |
| -------------------------------- | ---------------------------------- |
| 使用 `uv` 进行包管理             | "项目使用monorepo，包在 /packages" |
| 测试加 `--no-cache` 否则误报     | 目录结构描述                       |
| auth模块用自定义中间件，不要重构 | 技术栈描述                         |
| `legacy/` 已弃用但被三个模块导入 | 模块解释                           |

---

## 过滤规则

> **代理可以通过阅读代码自己发现这个吗？如果是，删除它。每一行都应代表仓库中尚不存在的信息。**

---

## 静态文件问题

| 问题                        | 说明                               |
| --------------------------- | ---------------------------------- |
| **静态但任务是动态**        | 扁平指令集无法根据任务类型条件判断 |
| **CSS重构不需要数据库警告** | 单体文件每次以相同方式加载         |

---

## 三层架构（推荐）

| 层级                  | 内容                                        |
| --------------------- | ------------------------------------------- |
| **第1层：协议文件**   | 路由文档——角色、技能、MCP、最小必要仓库事实 |
| **第2层：焦点文件**   | 根据任务类型有选择加载的上下文              |
| **第3层：维护子代理** | 保持协议文件准确性的智能体                  |

---

## 正确心态

| 视角                     | 做法                                             |
| ------------------------ | ------------------------------------------------ |
| **AGENTS.md 是诊断工具** | 添加一行，调查原因，修复底层问题，可能删除这一行 |
| **代码库问题优先**       | 在添加AGENTS.md前，先重构代码、添加linter规则    |
| **几乎空着开始**         | 添加指令："如果遇到令人困惑的东西，标记为评论"   |

---

## 关键结论

> **很多人本能想把 Coding Agents "完整入门一遍"，但对它们来说，真正稀缺的是边界条件、陷阱和禁区。与其反复重述仓库结构，不如把真正关键的约束写清楚。**
