# Vercel 如何构建 Coding Agents 的 AEO 追踪系统

> **作者**：Eric Dodds, Allen Zhou  
> **来源**：https://vercel.com/blog/how-we-built-aeo-tracking-for-coding-agents  
> **核心观点**：AI 引擎优化（AEO）需要追踪 Coding Agents 的搜索和引用行为

---

## 为什么需要 Coding Agents AEO

| 发现                           | 说明                                        |
| ------------------------------ | ------------------------------------------- |
| **20% 的提示触发网络搜索**     | 编程智能体频繁搜索相关文档                  |
| **搜索发生在真实开发工作流中** | 评估响应质量和来源准确性很重要              |
| **行为与纯模型不同**           | 需要完整开发环境：文件系统、Shell、包管理器 |

---

## Coding Agents AEO 六步生命周期

| 步骤                  | 动作                                              |
| --------------------- | ------------------------------------------------- |
| **1. 创建沙箱**       | 启动 MicroVM，配置运行时和超时                    |
| **2. 安装智能体 CLI** | 全局安装 npm 包（如 `@anthropic-ai/claude-code`） |
| **3. 注入凭证**       | 通过环境变量路由到 AI Gateway                     |
| **4. 运行智能体**     | 执行带提示的 Shell 命令                           |
| **5. 捕获记录**       | 提取工具调用、网络搜索、推荐内容                  |
| **6. 清理**           | 停止沙箱，确保无资源泄漏                          |

---

## 工具名称规范化

| 操作     | Claude Code  | Codex        | OpenCode | 规范名称     |
| -------- | ------------ | ------------ | -------- | ------------ |
| 读取文件 | `Read`       | `read_file`  | `read`   | `file_read`  |
| 写入文件 | `Write`      | `write_file` | `write`  | `file_write` |
| 编辑文件 | `StrReplace` | `patch_file` | `patch`  | `file_edit`  |
| 运行命令 | `Bash`       | `shell`      | `bash`   | `shell`      |
| 搜索网络 | `WebFetch`   | varies       | varies   | `web_fetch`  |

---

## 记录处理四阶段

| 阶段            | 功能                                                  |
| --------------- | ----------------------------------------------------- |
| **1. 记录捕获** | 特定于智能体（Claude读 JSONL文件，Codex/stdout 过滤） |
| **2. 解析**     | 规范化工具名称，扁平化消息结构                        |
| **3. 丰富**     | 提取元数据（URL、命令、路径）                         |
| **4. 摘要**     | 聚合统计，输入品牌提取管道                            |

---

## 关键发现

| 发现                       | 说明                                                     |
| -------------------------- | -------------------------------------------------------- |
| 编程智能体贡献大量搜索流量 | 优化内容对智能体很重要                                   |
| 推荐形状不同               | 智能体生成可执行代码（import、config、脚本），不只是散文 |
| 记录格式混乱               | 尽早构建规范化层避免持续崩溃                             |
| 相同管道适用模型和智能体   | 困难部分是上游：运行、捕获、规范化                       |

---

## 技术栈

| 工具                 | 用途                         |
| -------------------- | ---------------------------- |
| **Vercel Sandbox**   | 临时 MicroVM，执行隔离       |
| **AI Gateway**       | 统一路由，日志记录、成本追踪 |
| **Vercel Workflows** | 并行编排多个智能体           |

---

## 关键结论

> **AI 改变了人们查找信息的方式。对企业而言，理解 LLM 如何搜索和总结其网站内容至关重要。**
