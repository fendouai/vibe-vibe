# 你的 AI Coding Agents 需要一个管理者

> **作者**：Addy Osmani  
> **来源**：https://addyosmani.com/blog/coding-agents-manager/  
> **核心观点**：大规模 AI 编码是管理问题，不是提示问题

---

## 两种并行工作模式

| 模式                | 用途                                       |
| ------------------- | ------------------------------------------ |
| **本地高接触会话**  | 架构决策、棘手重构、产品细微差别           |
| **异步云/后台会话** | 专注有界任务：功能实现、测试生成、文档更新 |

---

## 管理技能的四个转移

| 技能                 | 管理场景                       | AI 编码场景                          |
| -------------------- | ------------------------------ | ------------------------------------ |
| **清晰任务范围界定** | 将意图转化为可执行的简报       | 写结果、上下文、约束、验收标准       |
| **委派**             | 决定谁拥有什么、多少自主权安全 | 机械实施委派，架构判断保留           |
| **验证循环**         | 早期反馈循环和质量门           | 要求 AI 运行测试、lint、结构化 PR 包 |
| **异步签到**         | 轻量稳定的状态汇报机制         | 15分钟无进展就停止并报告             |

---

## 委派边界

| 类型               | 示例                                         |
| ------------------ | -------------------------------------------- |
| **完全委派**       | 清晰规范的机械实施、样板重构、测试生成       |
| **委派但保持循环** | 共享接口、可能合并冲突、棘手边缘情况         |
| **不委派**         | 系统架构、跨领域重构、产品决策、安全关键设计 |

---

## 困难部分

| 问题                 | 解决方案                           |
| -------------------- | ---------------------------------- |
| **合并冲突快速倍增** | git worktrees + 每个代理一个 PR    |
| **品味成为瓶颈**     | "我们应该吗？"比"我们能吗？"更重要 |
| **WIP 限制**         | 限制活跃代理流数量，避免审查淹没   |

---

## 编排操作系统（循环）

| 步骤        | 动作                             |
| ----------- | -------------------------------- |
| **1. 规划** | 用结果、约束和验收标准写简报     |
| **2. 执行** | 把任务分配给边界清晰的并行智能体 |
| **3. 监控** | 轻量状态同步，快速解除阻碍       |
| **4. 验证** | 跑测试、lint、整理 PR 包         |
| **5. 集成** | 按深思熟虑的顺序合并             |
| **6. 复盘** | 更新 AGENTS.md 和检查清单        |

---

## 关键结论

> **高杠杆开发者会更像异步优先的管理者，手里同时跑着一小队并行的 AI Coding Agents。**
