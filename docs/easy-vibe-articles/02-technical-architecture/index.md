## 反直觉判断

> 很多时候，简单接口比专用框架更适合智能体

| 模式             | 价值                       |
| ---------------- | -------------------------- |
| 文件系统         | 承担大量状态管理工作       |
| Bash             | 重新变得重要               |
| 协议化上下文     | 接近模型擅长使用的系统表面 |
| 可观察的任务分工 | 更稳的执行框架             |

## 每个组件解决的问题

| 组件                 | 兜底问题                  |
| -------------------- | ------------------------- |
| 上下文怎么传         | 前置加载、system-reminder |
| 状态怎么留           | 文件系统、外部记忆        |
| 工具怎么暴露         | MCP 协议、Bash 接口       |
| 哪些能力该抽象成协议 | MCP、A2A                  |
| 哪些能力保留朴素接口 | 文件系统、Shell           |
| 什么时候协作         | 多智能体分工              |
| 什么时候隔离         | 沙箱执行、权限分级        |
