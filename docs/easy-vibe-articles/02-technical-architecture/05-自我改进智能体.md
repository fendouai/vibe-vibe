> **核心观点**：把反馈回路、评估机制与改进边界设计清楚

---

## 持续编码循环（Ralph Wiggum 技术）

```
选择下一个任务 → 实现任务 → 验证更改 → 提交代码 → 更新状态 → 重置上下文 → 下一轮
```

---

## 每轮迭代周期

| 步骤 | 操作                          |
| ---- | ----------------------------- |
| 1    | 选择下一个任务（从任务 JSON） |
| 2    | 实现任务                      |
| 3    | 运行测试、类型检查等验证      |
| 4    | 验证通过后提交                |
| 5    | 标记完成，记录学到内容        |
| 6    | 用新上下文进入下一轮          |

---

## 任务切分原则

| 原则         | 说明               |
| ------------ | ------------------ |
| **足够小**   | 一次 AI 会话能完成 |
| **标准明确** | 通过/失败标准清晰  |

**示例**：

| 错误             | 正确                                             |
| ---------------- | ------------------------------------------------ |
| "做完整个仪表板" | "新增导航栏，含首页、关于、联系链接，当前页高亮" |

## 编排循环伪代码

```bash
while :; do
   amp run -s prompt.md -o progress.txt
   if grep -q "<promise>COMPLETE</promise>" progress.txt; then break; fi
done
```

---

## AGENTS.md 手册

| 内容类型       | 说明                         |
| -------------- | ---------------------------- |
| **模式和约定** | 目录结构、组件组织、架构模式 |
| **陷阱**       | 易遗漏但遗漏就出错           |
| **风格与偏好** | 测试风格、代码风格、依赖原则 |
| **最近变更**   | 踩过的坑、修正和原因         |

> **条目要短、要准、要可执行**

---

## 内存持久化通道

| 文件             | 作用                              |
| ---------------- | --------------------------------- |
| **Git 提交历史** | git diff/git log 了解改了什么     |
| **progress.txt** | 每轮做了什么、成功/失败、错误信息 |
| **prd.json**     | 需求是否完成的状态                |
| **AGENTS.md**    | 跨任务复用的约束与经验            |

---

## 质量保证

| 检查           | 说明                         |
| -------------- | ---------------------------- |
| **单元测试**   | npm test / pytest            |
| **类型检查**   | tsc / MyPy / ESLint          |
| **模拟 CI**    | 本地把构建、测试、检查串起来 |
| **智能体验收** | UI 改动时启动浏览器检查      |

---

## 扩展策略

| 方式              | 说明                             |
| ----------------- | -------------------------------- |
| 不同范围          | 前端循环、后端循环、功能分支循环 |
| 规划者-工作者分层 | 高层智能体规划，下层智能体执行   |

> **单个循环做得足够稳、足够长，比一群并行循环更有价值**

---

## 监控与调试

| 工具             | 说明                         |
| ---------------- | ---------------------------- |
| **实时日志**     | 当前任务、测试结果、错误信息 |
| **检查点提交**   | git log/git diff 回看最近    |
| **任务状态概览** | 判断整体进度                 |
| **停止条件**     | 最大迭代数、最长运行时间     |

---

## 风险管理

| 防护措施         | 说明                         |
| ---------------- | ---------------------------- |
| **功能分支运行** | 不直接对 main下手            |
| **权限分级**     | 只读自动放行，写操作人工确认 |
| **沙箱运行**     | 容器或虚拟机执行             |
| **随时可中断**   | 知道怎么第一时间停掉         |

---

## 关键结论

> **自主编码智能体已经不再只是概念。只要流程设计得足够扎实，它们确实可以承担持续编码、验证和推进交付的工作**
