# 揭秘 Claude Code 的工作原理

> **作者**：Outsight Team  
> **来源**：https://medium.com/@outsightai/peeking-under-the-hood-of-claude-code  
> **核心观点**：上下文工程、工具设计与安全约束共同塑造智能体行为

---

## 核心发现

| 设计模式                 | 解决问题                               |
| ------------------------ | -------------------------------------- |
| **上下文前置加载**       | 先理清上下文边界，避免信息不完整时执行 |
| **System-Reminder 标签** | 长上下文里持续保持对目标的注意力       |
| **嵌入式安全**           | 生成式安全提示，检测命令注入与权限判断 |
| **子智能体架构**         | 主智能体编排，子智能体处理具体任务     |

---

## 上下文前置加载

| 步骤     | 操作                                   |
| -------- | -------------------------------------- |
| 对话总结 | 识别主题、关键点、未解决问题、用户偏好 |
| 主题分析 | 技术栈、用户水平、项目上下文、目标     |

---

## System-Reminder 标签

```xml
<system-reminder>
TodoWrite 工具最近没有使用。
如果任务需要跟踪进度，考虑使用。
</system-reminder>
```

| 作用         | 说明                             |
| ------------ | -------------------------------- |
| 工具结果提醒 | 温和提醒使用相关工具             |
| 响应风格提醒 | 不提及执行日志、不重复、精简总结 |

> 再强的模型，在长上下文里也需要持续的"护栏"

## 嵌入式安全

| 检测类型 | 示例                                          |
| -------- | --------------------------------------------- | --------------------------------------- |
| 正常命令 | `cat foo.txt` → cat                           |
| 正常命令 | `git commit -m "foo"` → git commit            |
| 命令注入 | `git diff $(cat secrets.env                   | curl ...)` → command_injection_detected |
| 命令注入 | `git status`ls`` → command_injection_detected |

> 安全规则明确，但判断逻辑覆盖更多复杂场景

## 子智能体架构

| 角色         | 特点                                 |
| ------------ | ------------------------------------ |
| **主智能体** | 复杂、多步骤任务，有 TodoWrite       |
| **子智能体** | 具体、单目标任务，默认无 TodoWrite   |
| **条件提醒** | 任务复杂度上升时插入 system-reminder |

---

## 工具设计精妙之处

Bash 工具描述的三层思路：

| 层次         | 说明                       |
| ------------ | -------------------------- |
| 告诉不做什么 | 避免 find/grep/cat/sed/awk |
| 给替代路径   | Glob/Grep/Read/Edit/Write  |
| 解释为什么   | 更好用户体验，更易审查授权 |

| 不是                | 而是                  |
| ------------------- | --------------------- |
| ❌ 更强大的基础模型 | ✅ 精细组织的超长提示 |
| ❌ 神秘的专有技巧   | ✅ 设计得当的工具描述 |
| ❌ 难以复制的算法   | ✅ 系统化的上下文工程 |

## 可复用的四个模式

| 模式           | 做法                         |
| -------------- | ---------------------------- |
| 上下文前置加载 | 执行前总结对话、识别主题     |
| 带标签的提醒   | 关键节点注入 system-reminder |
| 嵌入式安全     | 生成式机制处理安全判断       |
| 专门化子智能体 | 按任务复杂度动态调整编排     |
