# MCP：它是什么，以及为什么重要

> **作者**：Addy Osmani  
> **来源**：https://addyo.substack.com/p/mcp-what-it-is-and-why-it-matters  
> **核心观点**：让工具、数据源与模型连接拥有可复用的协议层

---

## MCP 是什么

> **模型上下文协议（MCP）** = AI 世界里的"USB-C"

| 定位         | 说明                                 |
| ------------ | ------------------------------------ |
| **通用插头** | 统一连接标准                         |
| **开放标准** | 让模型用一致方式连接不同应用与数据源 |
| **协议层**   | 把零散连接整理成统一接口             |

---

## 解决的问题

| 问题             | MCP 的解决                   |
| ---------------- | ---------------------------- |
| **碎片化集成**   | 每接一个工具都要重写适配     |
| **语言不匹配**   | 每个系统 API、参数、语义不同 |
| **供应商锁定**   | 一旦接上就难以切换           |
| **集成矩阵膨胀** | N 个工具 × M 个模型          |

> 过去 `N × M` 的集成矩阵 → 工具支持 MCP，平台支持 MCP

---

## MCP 架构

| 组成            | 职责                            |
| --------------- | ------------------------------- |
| **MCP 服务器**  | 贴近应用，暴露能力（翻译器）    |
| **MCP 客户端**  | AI 助手里连接服务器、喂回模型   |
| **MCP 协议**    | 消息格式、工具发现、错误处理    |
| **服务/数据源** | 真实目标（文件系统、SaaS、API） |

---

## MCP 的三类能力

| 类型     | 说明                               |
| -------- | ---------------------------------- |
| **资源** | 数据和内容（文件、数据库记录）     |
| **提示** | 可复用的提示模板                   |
| **工具** | 可执行的操作（API 调用、文件操作） |

---

## 真实用例

| 集成            | 说明                          |
| --------------- | ----------------------------- |
| **Figma MCP**   | AI 直接读取设计文件，生成代码 |
| **Blender MCP** | 自然语言控制 3D 建模          |
| **Unity MCP**   | 自然语言驱动游戏开发          |
| **Zapier MCP**  | 一次接入海量 SaaS 工具        |
| **GitHub MCP**  | 读仓库、开 issue、管理 PR     |
| **Slack MCP**   | 发消息、管理频道              |

## 最佳实践

| 实践             | 说明                       |
| ---------------- | -------------------------- |
| **工具描述清楚** | 做什么、何时用、参数含义   |
| **错误处理完整** | 给模型返回可行动的错误信息 |
| **权限最小化**   | 只读优先，收窄作用域       |
| **文档可上手**   | 安装方法、配置示例         |

---

## 当前挑战

| 挑战           | 说明                 |
| -------------- | -------------------- |
| 标准化身份验证 | 企业环境权限管理负担 |
| 生态成熟度     | 很多流行工具还没 MCP |
| 性能代价       | 多层通信增加延迟     |
| 安全隐私       | 权限大→误操作风险    |
| 配置门槛       | 非技术用户不够顺滑   |

---

## MCP 的未来

| 方向       | 说明                         |
| ---------- | ---------------------------- |
| 企业采用   | Salesforce、SAP 等官方服务器 |
| 开发者工具 | 调试器、生成器、监控面板     |
| 跨平台标准 | 模型厂商、客户端共同采用     |
| 智能体编排 | 多智能体协同工作             |

---

## 关键结论

> MCP 让 AI 与应用、数据和工具之间的交互，第一次有机会建立在统一、可复用、可扩展的协议层上
