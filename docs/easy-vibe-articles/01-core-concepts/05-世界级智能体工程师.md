# 如何成为世界级的智能体工程师

> **作者**：systematicls  
> **来源**：https://x.com/systematicls/status/2028814227004395561  
> **核心观点**：上下文管理、任务拆解、约束执行的能力决定产出

---

## 核心原则

> **上下文决定一切**

| 原则               | 说明                         |
| ------------------ | ---------------------------- |
| **少即是多**       | 过度热情可能弊大于利         |
| **精确控制上下文** | 只给 AI 任务所需的最精准信息 |
| **避免上下文膨胀** | 过多插件和依赖会淹没 AI      |

---

## 关键方法

### 1. 对实现细节保持精确

| 错误方式           | 正确方式                                     |
| ------------------ | -------------------------------------------- |
| "构建认证系统"     | "用 bcrypt + 7天过期 refresh token 实现认证" |
| AI 需要研究选项    | AI 确切知道你要什么                          |
| 上下文被可能性填满 | 上下文被具体实现填满                         |

### 2. 研究与实现分离

| 场景           | 处理方式                             |
| -------------- | ------------------------------------ |
| 你知道实现细节 | 直接让 AI 实现                       |
| 你不知道细节   | 先让 AI 研究，再决定，再让新 AI 实现 |

### 3. 使用中性 Prompt

| 错误方式               | 正确方式                             |
| ---------------------- | ------------------------------------ |
| "去数据库里找一个缺陷" | "梳理数据库逻辑，跟踪行为，汇报发现" |
| 推向预设结论           | 客观描述发现                         |

---

## 处理 AI 的讨好性

| AI 特性          | 应对策略                    |
| ---------------- | --------------------------- |
| AI 试图取悦用户  | 利用讨好性设计多 Agent 协作 |
| 要求找缺陷就会找 | 设置奖惩机制约束行为        |

**多 Agent 验证模式**：

| Agent       | 角色       | 约束           |
| ----------- | ---------- | -------------- |
| 寻找缺陷 AI | 找所有问题 | 按影响程度加分 |
| 对抗 AI     | 证伪缺陷   | 误判扣双倍分   |
| 裁判 AI     | 打分       | 说掌握真实答案 |

---

## 任务终止机制

| 方法              | 说明                           |
| ----------------- | ------------------------------ |
| **自动化测试**    | 测试通过才算完成，不许修改测试 |
| **截图验证**      | 测试通过后截图核对设计         |
| **Contract 文件** | 终止前必须完成的工作清单       |

---

## 长时间会话策略

| 做法                     | 问题       |
| ------------------------ | ---------- |
| 24 小时运行的单一会话    | 上下文膨胀 |
| 每个 Contract 一个新会话 | ✅ 推荐    |

> 设置编排层处理新 Contract，创建全新干净会话

---

## 迭代优化

| 持续动作        | 说明                        |
| --------------- | --------------------------- |
| **添加规则**    | AI 做了你不同意的事就写规则 |
| **添加 Skills** | 编码操作步骤，固化流程      |
| **定期清理**    | 合并去重，保持简洁          |

---

## Prompt 实用模式

### 分阶段任务分解

```
Phase 1: 分析现有代码结构
Phase 2: 设计实现方案
Phase 3: 实现核心逻辑
Phase 4: 添加测试和错误处理
Phase 5: 验证和优化
```

### 明确约束条件

```
Requirements:
- TypeScript strict mode
- 遵循现有命名规范
- 所有函数必须有 JSDoc
- 错误处理使用 Result<T, E> 模式
```

## 核心结论

| 原则         | 说明                          |
| ------------ | ----------------------------- |
| 上下文是王道 | 精确控制 AI 看到的信息        |
| 少即是多     | 避免过度配置和插件膨胀        |
| 迭代优化     | 持续调整规则和 Skills         |
| 明确终点     | 用测试和验证定义任务完成      |
| 对结果负责   | AI 不完美，你必须负责最终结果 |
