# 关于 Vibe Coding 你需要知道的一切

> **作者**：Zeb Hermann, Keith Messick (Vercel)  
> **来源**：https://vercel.com/blog/what-you-need-to-know-about-vibe-coding  
> **核心观点**：Vibe Coding 的本质是用更高层意图与 AI 协作完成实现

---

## 概念起源

> "屈服于感觉，拥抱指数级增长，忘记代码的存在。" — Andrej Karpathy

| 时间    | 事件                            |
| ------- | ------------------------------- |
| 2025.02 | Karpathy 提出 vibe coding 术语  |
| 现在    | >90% 美国开发者使用 AI 编程工具 |

---

## 从 No-code 到 Vibe Code

| 阶段          | 特点                 | 代表工具            |
| ------------- | -------------------- | ------------------- |
| **No-code**   | 所见即所得，拖拽组件 | 传统可视化工具      |
| **Vibe Code** | 所说即所得，描述需求 | v0、Lovable、Replit |

---

## 核心优势

| 优势         | 说明                               |
| ------------ | ---------------------------------- |
| **更快迭代** | 几天完成过去几个月的项目           |
| **更低成本** | 10 人 AI 团队 ≈ 100 传统工程师产出 |
| **门槛降低** | 真正门槛是"能否把需求说清楚"       |

---

## 用户画像

| 用户类型 | 占比 | 用途                     |
| -------- | ---- | ------------------------ |
| 非开发者 | 63%  | 简化工作流、构建定制方案 |
| 开发者   | 37%  | 快速原型、提高效率       |

---

## 安全护栏

| 需求     | 原因                         |
| -------- | ---------------------------- |
| 自动干预 | 不是每位用户都有安全训练背景 |
| 实时扫描 | 阻断漏洞、紧急停止           |
| 部署拦险 | 高风险问题在部署阶段拦截     |

> **最成功的平台，把安全能力直接做进界面和默认流程**

---

## 关键要点

| 要点              | 说明                      |
| ----------------- | ------------------------- |
| 不是外包代码给 AI | 学会用高层意图与它协作    |
| 前提条件成立      | 需要清晰的需求描述        |
| 适合特定任务      | 原型、个人项目、快速验证  |
| 不是无需理解系统  | "自然语言编程"≠"无需理解" |

---

## 企业选择

| 选择             | 说明                                |
| ---------------- | ----------------------------------- |
| 拥抱 Vibe Coding | 更精简团队更快交付                  |
| 继续传统流程     | 关键系统约束更强                    |
| 混合模式         | 低风险环节用 AI，高风险环节保持传统 |
