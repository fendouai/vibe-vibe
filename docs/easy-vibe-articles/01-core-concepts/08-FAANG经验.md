# 我们在 FAANG 是怎么做 Vibe Coding 的

> **作者**：Anonymous FAANG Engineer  
> **来源**：https://www.reddit.com/r/ClaudeAI/comments/1i4kqxo/how_we_vibe_code_at_a_faang/  
> **核心观点**：把规格、测试与部署流程一起纳入协作

---

## 完整流程

| 步骤                | 内容                                        |
| ------------------- | ------------------------------------------- |
| **1. 技术设计文档** | 提案 → 方向认可 → 系统设计 → 架构/边界/依赖 |
| **2. 设计审查**     | 资深工程师挑薄弱点，尽早暴露问题            |
| **3. 子系统文档**   | 各团队负责模块，写清楚接口/职责/交付边界    |
| **4. 冲刺规划**     | PM/TPM 拆成离散任务，明确优先级和依赖       |
| **5. 软件开发**     | 测试驱动 → 先补测试 → 再参与实现            |
| **6. 代码审查**     | 至少两位开发者批准才能合并                  |
| **7. 暂存验证**     | 合并后暂存环境验证 → 正常才推生产           |

---

## AI 的价值位置

| 环节     | AI 作用        |
| -------- | -------------- |
| 技术设计 | 协助完善文档   |
| 开发实现 | 显著加快节奏   |
| 测试编写 | 补上测试再实现 |
| 代码审查 | 参与辅助审查   |

> **前提：流程本身已经清晰**

---

## 测试驱动开发

| 步骤 | 操作                              |
| ---- | --------------------------------- |
| 1    | 先让 Coding Agents 为功能补上测试 |
| 2    | 再让它们参与实现                  |
| 3    | 任务边界清楚，验收标准明确        |

---

## 总体效果

| 指标               | 提升             |
| ------------------ | ---------------- |
| 功能提案到上线周期 | 缩短约 30%       |
| 高要求团队         | 已是非常可观提升 |

---

## TL;DR

> **先把设计文档和架构想清楚，再按模块推进；先写测试，再做实现。AI 真正放大的是一套严谨流程，而不是取代这套流程。**
