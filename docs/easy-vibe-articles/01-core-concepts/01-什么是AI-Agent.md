# 什么是 AI Agent？为什么它们重要？

> **作者**：Addy Osmani  
> **来源**：https://addyo.substack.com/p/what-are-ai-agents-why-do-they-matter  
> **核心观点**：Agent 是能维持上下文、调用工具并连续完成任务的工作系统

---

## 核心定义

| 类型         | 特点                           | 能力                         |
| ------------ | ------------------------------ | ---------------------------- |
| **AI Model** | 单次交互、被动响应、无状态     | 提供输入 → 返回输出          |
| **AI Agent** | 自主系统、维持上下文、连续执行 | 感知环境 → 做决策 → 采取行动 |

---

## Agent 的核心能力

| 能力              | 说明                               |
| ----------------- | ---------------------------------- |
| **持久上下文**    | 在多次交互中保持记忆，从经验中学习 |
| **工具使用**      | 通过 API、浏览器、系统交互执行操作 |
| **任务分解**      | 将复杂目标拆成可管理的子任务       |
| **自我监控**      | 监控进度，根据结果调整策略         |
| **多 Agent 协作** | 每个 Agent 专门负责不同方面        |

---

## 标准化协议

| 协议    | 定位          | 作用                        |
| ------- | ------------- | --------------------------- |
| **MCP** | Agent → 工具  | 连接外部工具和数据源        |
| **A2A** | Agent → Agent | 不同供应商 Agent 之间的协作 |

> MCP 是"AI 集成的 USB-C"，A2A 是 Agent 之间的通信标准

---

## Agent 架构模式

### 1. 工具使用与集成

- 何时使用哪些工具
- 如何有效组合多个工具
- 编排连贯的工具调用链

### 2. 记忆与上下文管理

| 记忆类型 | 作用                       |
| -------- | -------------------------- |
| 情景记忆 | 过去交互及其结果           |
| 工作记忆 | 当前任务和最近行动的上下文 |
| 语义记忆 | 长期学习的事实、模式和关系 |

### 3. 分层规划与执行

| 层级     | 职责               |
| -------- | ------------------ |
| 战略规划 | 高层目标设定       |
| 战术规划 | 策略分解为任务     |
| 执行规划 | 每个任务的具体步骤 |
| 行动执行 | 执行并监控结果     |

### 4. 多 Agent 系统

| 角色         | 职责               |
| ------------ | ------------------ |
| 协调器 Agent | 管理任务流程和委派 |
| 专家 Agent   | 特定领域深厚知识   |
| 评论家 Agent | 审查和验证工作     |
| 集成 Agent   | 处理 Agent 间通信  |

## 浏览器 Agent 生态

| 类型         | 代表产品                          |
| ------------ | --------------------------------- |
| **专有方案** | Google Mariner、OpenAI Operator   |
| **开源方案** | Browser Use、Skyvern、Browserbase |

> 浏览器 Agent 用于"自动化成本太高"的长尾任务

---

## 现实应用场景

| 领域       | 应用                             |
| ---------- | -------------------------------- |
| 软件开发   | 代码生成、审查、测试、文档维护   |
| 企业自动化 | 文档管理、客户服务、工作流自动化 |
| 研究任务   | 跨多个网站收集信息并综合报告     |

---

## 当前挑战

| 挑战         | 说明                         |
| ------------ | ---------------------------- |
| **可靠性**   | 幻觉、错误信心、错误传播     |
| **安全隐私** | 访问控制、数据隐私、审计问责 |
| **经济成本** | 计算资源、集成开销、技能要求 |

---

## 关键结论

> **未来十年将由我们与 AI 协作的程度来定义——而不仅仅是它有多聪明**

- Agent 不是取代开发者，而是扩展开发者能力
- 成功采用的关键在于理解系统的潜力和局限性
- 把方向盘保持在人手里
