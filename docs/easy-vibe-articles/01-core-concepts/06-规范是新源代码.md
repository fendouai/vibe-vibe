# 规范是新的源代码

> **作者**：Ravi Mehta & Danny Martinez  
> **来源**：https://blog.ravi-mehta.com/p/specs-are-the-new-source-code  
> **核心观点**：规范从"可选文档"升级为关键生产资产

---

## 核心转变

| 传统模式                 | AI 时代模式              |
| ------------------------ | ------------------------ |
| 源代码 → 编译 → 目标代码 | 规范 → AI → 代码         |
| 源代码是真实来源         | 规范是真实来源           |
| 保留代码，丢弃提示       | 保留规范，代码是有损投影 |

---

## 为什么 PM 成为瓶颈

| 原因                   | 说明                                  |
| ---------------------- | ------------------------------------- |
| 工程师交付速度提高数倍 | Andrew Ng：看到 2倍 PM/工程师比例提案 |
| 产品交付加速           | 对理解需求、验证影响施加压力          |
| 压力集中在规范         | 规范成为关键工件                      |

---

## 规范的新定位

> "一个足够强大的规范可以生成好的 TypeScript、Rust、服务器、客户端、文档、教程、博客文章，甚至播客。" — Sean Grove

| 属性               | 说明                           |
| ------------------ | ------------------------------ |
| **对齐人类和机器** | 共同目标的工件                 |
| **包含一切**       | 意图、约束、边界条件           |
| **优于代码**       | 代码是有损投影，规范是完整意图 |

---

## 原型没有杀死规范

| 旧工作流                              | 新工作流                                        |
| ------------------------------------- | ----------------------------------------------- |
| 模糊想法 → 线框图 → 设计 → MVP → 反馈 | 模糊想法 → 快速原型 → 反馈 → 清晰规范 → AI 实施 |
| 几周时间                              | 几小时                                          |
| 需要工程                              | 无需工程                                        |

> **原型让规范变得更好，而不是取代规范**

## 规范驱动开发实战

### 流程步骤

| 步骤 | 操作                       |
| ---- | -------------------------- |
| 1    | Slack 消息生成 Linear 工单 |
| 2    | 澄清想要的具体内容         |
| 3    | Claude 打开工单并审查      |
| 4    | Claude 分析相对于代码库    |
| 5    | 创建分支并实施             |
| 6    | 测试确保按预期工作         |
| 7    | 打开 PR 等待审查           |

### 成效条件

| 条件       | 说明                               |
| ---------- | ---------------------------------- |
| **具体化** | 模糊规范 → 混乱代码库              |
| **选择性** | 简单任务适用，复杂任务需要专业人士 |
| **守门**   | 实际懂的人在审查并确保平衡         |

## AI 收益的不均匀分布

| 运行速度     | 说明                           |
| ------------ | ------------------------------ |
| **AI 速度**  | 生成代码、文本、图像           |
| **人类速度** | 与客户交谈、发现需求、说服购买 |

> 聚光灯从实施工作转移到理解工作

## 关键结论

> **最佳 PM 将洞察转化为规范——对齐团队、指导实施、作为持久工件**
