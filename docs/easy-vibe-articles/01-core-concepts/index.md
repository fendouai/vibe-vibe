## 核心概念对照

| 概念                    | 定义                             | 关键差异                       |
| ----------------------- | -------------------------------- | ------------------------------ |
| **AI Model**            | 单次交互，被动响应，无状态       | 每个响应孤立存在               |
| **AI Agent**            | 自主系统，维持上下文，连续执行   | 能感知环境、做出决策、采取行动 |
| **Vibe Coding**         | 顺着感觉推进，不把代码审查放优先 | 快速原型，适合周末项目         |
| **Agentic Engineering** | AI 负责实现，人把控架构与质量    | 有约束的协作流程               |

---

## 开发者角色迁移

| 传统角色         | 新角色               | 变化本质              |
| ---------------- | -------------------- | --------------------- |
| 亲手实现每个细节 | 定义目标、组织上下文 | 从执行者到决策者      |
| 编写代码         | 拆解任务、审查结果   | 从 Coder 到 Commander |
| 关注实现细节     | 对系统质量负责       | 从手工制作到编排系统  |

---

## 关键判断

> **快，不等于随便**

- AI 压低了实现门槛
- 也同步放大了规格不清、验证不足和质量纪律松动的代价
- 真正拉开差距的是：组织上下文、约束任务、管理工作流的能力
