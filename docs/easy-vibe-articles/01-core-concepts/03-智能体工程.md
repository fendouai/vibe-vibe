# 智能体工程（Agentic Engineering）

> **作者**：Addy Osmani  
> **来源**：https://addyosmani.com/blog/agentic-engineering/  
> **核心观点**：把计划、审查、测试和责任重新拉回开发主线

---

## 与 Vibe Coding 的区别

| 维度         | Vibe Coding      | Agentic Engineering |
| ------------ | ---------------- | ------------------- |
| **核心特点** | 顺着感觉快速推进 | 有约束的协作流程    |
| **代码审查** | 后置或不做       | 始终优先            |
| **适用场景** | 周末项目、原型   | 严肃工程流程        |
| **责任归属** | 模糊             | 人对架构、质量负责  |

---

## 智能体工程的工作流

| 步骤         | 操作                               |
| ------------ | ---------------------------------- |
| **先计划**   | 写设计文档或规范（可 AI 协助完成） |
| **先指导**   | 把范围明确的任务交给 AI            |
| **再审查**   | 像审查团队成员 PR一样审查输出      |
| **测试为主** | 稳定的验证闭环是关键               |

---

## 核心原则

> 如果一段实现你自己都无法解释清楚，就不该直接放进代码库

| 原则               | 说明                         |
| ------------------ | ---------------------------- |
| 先写规范再给任务   | 把工作拆成边界清晰的任务     |
| AI 负责实现推进    | 人负责架构、质量与正确性     |
| 测试不是补丁       | 是把协作流程变得可靠的关键   |
| 始终由你拥有代码库 | 文档、版本控制、CI、可观测性 |

---

## 技能差距

| 经验层级       | 影响                                 |
| -------------- | ------------------------------------ |
| **资深工程师** | AI 放大优势，快速判断什么是好实现    |
| **基础不稳**   | 风险明显，产出功能但不理解交付了什么 |

> 智能体工程没有让工程"更容易"，只是把难点换了位置

---

## 新的职责

| 减少的工作 | 增加的工作   |
| ---------- | ------------ |
| 手工敲代码 | 定义任务     |
| -          | 组织上下文   |
| -          | 审查结果     |
| -          | 建立验证机制 |

---

## 关键结论

> **氛围编程展示了完全放开约束时会发生什么。现在，是时候把"工程"重新放回中心了。**
