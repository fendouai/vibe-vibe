# 14.1 系统设计原则

## 量化系统的设计目标

构建量化交易系统时，需要平衡以下目标：

| 目标     | 说明                   | 优先级     |
| -------- | ---------------------- | ---------- |
| 可靠性   | 系统稳定运行，不丢单   | ⭐⭐⭐⭐⭐ |
| 可扩展性 | 容易增加新策略、新市场 | ⭐⭐⭐⭐   |
| 可维护性 | 代码清晰，易于修改     | ⭐⭐⭐⭐   |
| 性能     | 低延迟，高吞吐         | ⭐⭐⭐     |
| 安全性   | 防止错误和攻击         | ⭐⭐⭐⭐⭐ |

## 核心设计原则

### 1. 单一职责原则

每个组件只做一件事：

```
✅ 好设计：
DataFeed → Strategy → RiskManager → OrderRouter → BrokerAPI

❌ 坏设计：
超级类TradingSystem (包含数据、策略、风控、下单、可视化...)
```

```python
# 单一职责示例
class DataFeed:
    """只负责：获取和提供市场数据"""
    def connect(self): ...
    def subscribe(self, symbol): ...
    def get_price(self, symbol): ...

class Strategy:
    """只负责：根据数据生成交易信号"""
    def generate_signal(self, market_data): ...

class RiskManager:
    """只负责：风控检查和限制"""
    def check_order(self, order): ...

class OrderRouter:
    """只负责：订单路由和执行"""
    def submit_order(self, order): ...
```

### 2. 事件驱动架构

组件之间通过事件通信，降低耦合：

```python
class EventBus:
    """事件总线"""
    def __init__(self):
        self.subscribers = defaultdict(list)

    def subscribe(self, event_type, handler):
        self.subscribers[event_type].append(handler)

    def publish(self, event):
        for handler in self.subscribers.get(event.type, []):
            handler(event)

# 使用事件总线解耦
event_bus = EventBus()

# 策略订阅市场数据
def on_market_data(data):
    signal = strategy.generate_signal(data)
    event_bus.publish(Event('SIGNAL', signal))

data_feed.on_market_data = on_market_data

# 风控订阅信号
def on_signal(event):
    if not risk_manager.check(event.data):
        return
    event_bus.publish(Event('ORDER', event.data))

event_bus.subscribe('SIGNAL', on_signal)
```

### 4. 错误处理和容错

```python
class ErrorHandler:
    """统一错误处理"""

    @staticmethod
    def handle_exception(func):
        """装饰器：自动捕获和处理异常"""
        def wrapper(*args, **kwargs):
            try:
                return func(*args, **kwargs)
            except ConnectionError as e:
                ErrorHandler.retry_connection(func, args, kwargs)
            except OrderRejectError as e:
                ErrorHandler.log_and_alert('ORDER_REJECT', str(e))
            except Exception as e:
                ErrorHandler.handle_unknown_error(e)
        return wrapper

    @staticmethod
    def retry_connection(func, args, kwargs, max_retries=3):
        """重试机制"""
        for i in range(max_retries):
            try:
                return func(*args, **kwargs)
            except ConnectionError:
                time.sleep(2 ** i)  # 指数退避
        raise ConnectionError(f"重试{max_retries}次后仍然失败")

# 使用
@ErrorHandler.handle_exception
def place_order(order):
    broker.submit_order(order)
```

### 5. 日志和监控

```python
import logging
from logging.handlers import RotatingFileHandler

def setup_logger(name, log_file):
    logger = logging.getLogger(name)
    logger.setLevel(logging.INFO)

    handler = RotatingFileHandler(log_file, maxBytes=10*1024*1024, backupCount=5)
    formatter = logging.Formatter(
        '%(asctime)s | %(levelname)-8s | %(name)s | %(message)s'
    )
    handler.setFormatter(formatter)
    logger.addHandler(handler)

    return logger

# 创建日志器
system_logger = setup_logger('system', 'logs/system.log')
trade_logger = setup_logger('trades', 'logs/trades.log')
risk_logger = setup_logger('risk', 'logs/risk.log')

# 使用
system_logger.info("系统启动")
trade_logger.info(f"买入 {symbol} {quantity} @ {price}")
risk_logger.warning(f"风控拒绝订单: {reason}")
```

## 扩展性设计

```python
# 策略注册机制
class StrategyRegistry:
    """策略注册表"""
    _strategies = {}

    @classmethod
    def register(cls, name):
        def decorator(strategy_class):
            cls._strategies[name] = strategy_class
            return strategy_class
        return decorator

    @classmethod
    def get(cls, name):
        return cls._strategies.get(name)

    @classmethod
    def list_strategies(cls):
        return list(cls._strategies.keys())

# 注册策略
@StrategyRegistry.register('momentum')
class MomentumStrategy(Strategy):
    ...

@StrategyRegistry.register('mean_reversion')
class MeanReversionStrategy(Strategy):
    ...

# 动态加载策略
def create_strategy(config):
    strategy_name = config['name']
    strategy_class = StrategyRegistry.get(strategy_name)

    if strategy_class is None:
        raise ValueError(f"Unknown strategy: {strategy_name}")

    return strategy_class(**config['params'])
```

## 总结

系统设计的核心原则：

| 原则     | 说明           | 实践              |
| -------- | -------------- | ----------------- |
| 单一职责 | 组件职责清晰   | 每个类不超过200行 |
| 事件驱动 | 松耦合         | 使用EventBus通信  |
| 配置分离 | 代码与配置分离 | 使用YAML配置      |
| 错误处理 | 容错机制       | 重试、降级、告警  |
| 日志监控 | 可观测性       | 结构化日志        |
| 易于测试 | 模块可测       | 依赖注入          |
