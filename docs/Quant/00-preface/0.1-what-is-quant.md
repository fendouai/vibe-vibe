# 0.1 量化交易的定义：从主观交易到系统化交易

## 什么是量化交易

**量化交易（Quantitative Trading）** 是指使用数学模型、统计分析和计算机算法来制定交易决策的交易方式。

量化交易的核心是**系统化**——把人的交易想法，变成清晰的、可执行的规则。

```
传统主观交易：
"我觉得这只股票跌得差不多了，应该会反弹"

量化交易：
IF 股价 < 20日均线 AND RSI < 30 AND 成交量 > 过去20日平均成交量
THEN 买入 100 股
```

## 量化交易 vs 主观交易

| 维度     | 主观交易       | 量化交易       |
| -------- | -------------- | -------------- |
| 决策方式 | 依赖人的判断   | 依赖数学规则   |
| 执行速度 | 慢，受情绪影响 | 快，毫秒级执行 |
| 策略容量 | 有限           | 可规模化       |
| 可验证性 | 难以回溯验证   | 可历史回测     |
| 纪律性   | 易受情绪干扰   | 严格执行规则   |

## 量化交易的类型

### 1. 趋势跟随 (Trend Following)

- **原理**：顺势而为，当价格朝某方向运行时，跟随趋势
- **经典指标**：移动平均线、MACD、ADX
- **代表人物**：Richard Dennis（海龟交易法则）

### 2. 均值回归 (Mean Reversion)

- **原理**：价格偏离均值后会回归
- **经典指标**：布林带、RSI、Z-Score
- **适用场景**：震荡市场

### 3. 配对交易 (Pairs Trading)

- **原理**：两个相关资产价差偏离后会回归
- **方法**：协整检验、统计套利
- **优势**：市场中性

### 4. 统计套利 (Statistical Arbitrage)

- **原理**：利用数学模型发现价格偏差
- **方法**：机器学习、多因子模型
- **代表机构**：Citadel、D.E. Shaw

## 量化交易的工作流程

```
1. 想法生成 (Idea Generation)
   ↓ 发现一个市场规律或交易想法
2. 数据研究 (Data Research)
   ↓ 收集数据，验证想法
3. 策略开发 (Strategy Development)
   ↓ 把想法变成具体规则
4. 回测验证 (Backtesting)
   ↓ 用历史数据验证策略效果
5. 风险管理 (Risk Management)
   ↓ 确定头寸、风控规则
6. 实盘执行 (Live Trading)
   ↓ 连接经纪商，自动化执行
7. 监控优化 (Monitoring)
   ↓ 持续监控，迭代改进
```

## 量化交易的关键要素

### 数据

- 历史价格数据（开盘价、最高价、最低价、收盘价、成交量）
- 财务报表数据
- 宏观经济数据
- 替代数据（舆情、卫星图像等）

### 策略

- 入场规则：什么时候买
- 出场规则：什么时候卖
- 仓位管理：买多少

### 执行

- 经纪商 API
- 订单管理系统
- 延迟优化

### 风控

- 单笔交易最大亏损
- 每日最大亏损
- 组合风险限制
