# 11.1 券商API概述

## 什么是券商API？

券商API（Broker API）是券商提供给量化交易者的编程接口，允许程序化地下单、查询账户、获取市场数据等。相比手动交易，API交易具有以下优势：

- **速度**: 毫秒级下单
- **一致性**: 消除人为错误
- **自动化**: 可以24小时运行
- **可扩展**: 同时管理多个账户

## 主流券商API对比

| 券商                | 市场       | API类型         | 费用     | 特点               |
| ------------------- | ---------- | --------------- | -------- | ------------------ |
| Alpaca              | 美股、加密 | REST, WebSocket | 免费     | 简单易用，适合入门 |
| Interactive Brokers | 全球       | TWS API, REST   | 低佣金   | 功能强大，复杂     |

| Binance             | 加密货币   | REST, WebSocket | 低手续费 | 币种丰富           |
| Coinbase            | 加密货币   | REST            | 中等     | 合规可靠           |

## WebSocket实时数据

```python
# WebSocket连接示例结构
import websockets
import json

async def connect_market_data():
    uri = "wss://stream.data.alpaca.markets/v2/iex"

    async with websockets.connect(uri) as ws:
        # 认证
        await ws.send(json.dumps({
            "action": "auth",
            "key": "YOUR_API_KEY",
            "secret": "YOUR_API_SECRET"
        }))

        # 订阅数据
        await ws.send(json.dumps({
            "action": "subscribe",
            "trades": ["AAPL", "TSLA"],
            "quotes": ["AAPL", "TSLA"],
            "bars": ["AAPL", "TSLA"]
        }))

        # 接收数据
        async for message in ws:
            data = json.loads(message)
            print(data)
```

### 3. 错误处理

```python
import requests
from requests.exceptions import RequestException

def safe_order(order_params, max_retries=3):
    """带重试机制的下单函数"""
    for attempt in range(max_retries):
        try:
            response = requests.post(
                f"{BASE_URL}/v2/orders",
                json=order_params,
                headers={
                    "APCA-API-KEY-ID": API_KEY,
                    "APCA-API-SECRET-KEY": API_SECRET
                }
            )

            if response.status_code == 200:
                return response.json()
            elif response.status_code == 429:
                # 请求过多，等待后重试
                import time
                time.sleep(2 ** attempt)
            else:
                print(f"Error: {response.status_code} - {response.text}")
                return None

        except RequestException as e:
            print(f"Request failed: {e}")
            if attempt == max_retries - 1:
                raise
            import time
            time.sleep(1)

    return None
```

## API速率限制

| 券商    | 限制          | 说明     |
| ------- | ------------- | -------- |
| Alpaca  | 200 req/min   | 账户级别 |
| Binance | 1200 req/min  | IP级别   |
| IB      | 50 req/second | 账户级别 |

```python
import time
from collections import deque

class RateLimiter:
    """简单的速率限制器"""
    def __init__(self, max_requests, time_window):
        self.max_requests = max_requests
        self.time_window = time_window
        self.requests = deque()

    def wait_if_needed(self):
        now = time.time()

        # 移除过期的请求记录
        while self.requests and self.requests[0] < now - self.time_window:
            self.requests.popleft()

        if len(self.requests) >= self.max_requests:
            # 需要等待
            sleep_time = self.time_window - (now - self.requests[0])
            if sleep_time > 0:
                time.sleep(sleep_time)

        self.requests.append(time.time())
```

## 连接测试

```python
import requests

def test_api_connection(base_url, api_key, api_secret):
    """测试API连接是否正常"""
    try:
        response = requests.get(
            f"{base_url}/v2/account",
            headers={
                "APCA-API-KEY-ID": api_key,
                "APCA-API-SECRET-KEY": api_secret
            }
        )

        if response.status_code == 200:
            account = response.json()
            print("✅ API连接成功!")
            print(f"   账户ID: {account.get('id', 'N/A')}")
            print(f"   账户价值: ${float(account.get('portfolio_value', 0)):.2f}")
            print(f"   状态: {account.get('status', 'N/A')}")
            return True
        else:
            print(f"❌ API连接失败: {response.status_code}")
            print(f"   {response.text}")
            return False

    except Exception as e:
        print(f"❌ 连接异常: {e}")
        return False

# 测试连接
test_api_connection("https://paper-api.alpaca.markets",
                    "YOUR_KEY", "YOUR_SECRET")
```

## 选择券商的考虑因素

| 因素     | 问题                       |
| -------- | -------------------------- |
| 市场覆盖 | 你想交易哪些市场？         |
| 费用     | 佣金和平台费是多少？       |
| API质量  | 文档是否完善？稳定性如何？ |
| 订单类型 | 支持哪些订单类型？         |
| 提款     | 资金转入转出方便吗？       |
| 监管     | 是否受正规机构监管？       |

## 总结

1. **券商API是量化交易的核心工具**，实现从研究到执行的桥梁

4. **处理错误和限制**，API不是100%可靠的
