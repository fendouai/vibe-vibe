# 基础技术与架构设计

## 编者按

理念很美好，但魔鬼在细节里。

当我们谈论"让 AI 自主完成任务"时，背后是一整套技术架构的支撑：智能体如何推理和规划？如何记住上下文？如何调用工具和感知环境？这些问题的答案，决定了你的智能体是"能用"还是"好用"。

Vercel 的工程实践给出了一个令人意外的答案：**简单比复杂更有效**。

他们用文件系统替代复杂的状态管理，用 Bash 替代专门的工具框架，成本降低 75%，性能反而提升。这不是偷懒，而是深刻理解了智能体的本质：AI 最擅长的是理解自然的、通用的接口，而不是学习复杂的、专用的 API。

本专栏收录的文章展示了两个关键洞察：

- **文件系统 + Bash 的威力**：最古老的 Unix 哲学，在 AI 时代焕发新生
- **严格的实验方法论**：不是拍脑袋决策，而是用 A/B 测试验证每个假设

这些文章不会教你使用某个具体框架，而是教你如何思考架构设计——什么时候该简化，什么时候该复杂化，如何在理论和实践之间找到平衡。
