# 工具链与开发框架

## 编者按

工具的选择，暴露了你对问题的理解深度。

在 Agentic Coding 的早期，我们本能地认为"工具越多越好"——给 AI 更多能力，它就能做更多事情。但 Vercel 的实践证明，这是个陷阱。

他们的 d0 智能体最初有 50+ 个工具，性能糟糕。经过反复实验，他们删掉了 80% 的工具，只保留 10 个核心工具，性能反而提升了 3.5 倍。原因很简单：**工具越多，AI 的决策负担越重，出错概率越高**。

更激进的发现是：**被动上下文（AGENTS.md）比主动检索（Skills）更可靠**。与其让 AI 主动搜索信息，不如直接把信息放在它面前。这违反直觉，但符合 AI 的工作原理——大语言模型擅长理解上下文，而不是执行复杂的检索逻辑。

本专栏收录的文章揭示了工具选择的两个核心原则：

- **少即是多**：精简工具集，降低决策复杂度
- **被动优于主动**：用配置文件替代动态检索

这些不是理论推导，而是 Vercel 在生产环境中用真实数据验证的结论。如果你正在构建自己的智能体工具链，这些经验能帮你避开最大的坑。
