# 第 2 章：带回自己的工作台 - 从平台到本地

---

## 本章目标

**把 v1 原型带回你自己的电脑，让它变成可长期修改的项目**

- 项目在本地跑起来
- 知道关键文件大概在哪里
- 完成一次最小修改

---

## 第一步：导出项目到本地（5 分钟）

### 为什么要带回本地

| 你会得到什么 | 为什么重要                             |
| ------------ | -------------------------------------- |
| 固定位置     | 知道项目以后放在哪里，不会每次都重新找 |
| 长期可改     | 后面每一轮优化都有稳定落点             |
| 文件可见     | 知道页面、样式、聊天代码大概在哪里     |
| 更可控       | 报错、改动、存档都围绕同一个项目进行   |

### 导出操作顺序

1. 找平台的"导出代码"、"下载项目"或"同步代码"入口
2. 下载当前可用版本（不是旧草稿）
3. 解压（如果是压缩包）
4. **重命名文件夹**（很重要！）

### 文件夹命名建议

✅ 好的命名：

- `my-homepage`
- `digital-twin-site`
- `linan-homepage`
- `my-site-v1`

❌ 避免的命名：

- `新建文件夹`
- `项目最终版`
- `项目最终版2`
- `个人主页最新最终不改了`

**放到固定位置**：`~/Projects/`、`~/Code/`、`桌面/我的项目/`

---

## 第二步：用 AI IDE 打开项目（15 分钟）

### 只选一个工具

不要做 AI IDE 大横评，选一个你现在最方便用的就够了。

- Cursor
- Trae
- Claude Code
- 其他你熟悉的编辑器

**决定标准**：谁能让我今天把项目打开并跑起来

### 让 AI 陪你启动项目

直接对 AI 说：

```text
我刚把一个个人主页项目导出到本地。

请先帮我判断这是个什么项目，告诉我应该怎么在本地跑起来。
如果需要我执行命令，请一步一步告诉我。
如果报错了，请根据报错继续带我排查。
```

### 启动流程

1. 用 AI IDE 打开项目文件夹
2. 让 AI 判断项目类型和启动方式
3. 按 AI 给的步骤安装依赖、运行项目
4. 在浏览器打开本地地址，确认页面跑起来

### 如果报错

**不要自己猜**，把完整报错交给 AI：

```text
这是我刚刚运行时的完整报错。

请先判断最可能的原因是什么，
再告诉我下一步只做什么，不要一次给太多方案。
```

**记住**：先判断，再行动，一次只改一个变量

---

## 第三步：认识项目地图（10 分钟）

### 只要知道三类东西在哪里

| 你要找什么   | 为什么先找它                         |
| ------------ | ------------------------------------ |
| 页面入口     | 以后很多内容和布局调整，都从这里开始 |
| 聊天相关代码 | 数字分身的展示和交互，通常会落在这里 |
| 样式或配置   | 改配色、间距、文案时，经常会碰到     |

### 让 AI 用人话讲结构

```text
请你先用人话介绍这个项目的结构，告诉我：
1. 主页入口大概在哪
2. 数字分身聊天区相关代码大概在哪
3. 样式主要在哪些文件里
4. 现阶段我最不需要先读懂的部分是什么

不要展开太深，只给我一张够用地图。
```

**目标**：建立"我不用懂每一行，但我知道该去哪里找"的感觉

---

## 第四步：完成一次最小修改（10 分钟）

### 第一次修改，越小越好

推荐改这些最容易被看到变化的内容：

- 改一句欢迎语
- 改一段自我介绍
- 改一个按钮文案
- 改聊天区引导语

### 为什么要这么小？

练的不是产出规模，而是流程感：

> 我提出修改 → AI 帮我改 → 本地页面变化 → 我确认结果

### 一个够用的第一次任务

```text
请帮我把首页的一句话介绍改得更像我本人。
目标文案是：一个正在学习用 AI 做产品的内容策划。

请告诉我这句文案在哪个文件里，
并只修改这一处，不要动其他布局和样式。
```

这句话同时具备三样东西：**目标 + 范围 + 约束**

---

## 三类高频提问（后面会一直用）

### 1. 理解类：这个项目结构在做什么？

```text
请用人话解释这个项目目前的结构。
告诉我主页入口、聊天相关代码、样式相关文件分别大概在哪。
不要展开太深。
```

### 2. 定位类：我想改 X，代码在哪？

```text
我想修改首页首屏的一句话介绍和主按钮文案。

请先告诉我这两处分别在哪个文件里，再给出修改建议。
```

### 3. 约束类：只改这一处，不要动其他地方

```text
请只修改聊天框的引导语，
不要改布局，不要新增模块，也不要调整配色。
```
