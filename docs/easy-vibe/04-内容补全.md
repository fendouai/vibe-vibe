# 第 4 章：让主页更完整 - 内容补全与 Git 闭环

---

## 本章目标

**让页面不再只是好看的空壳，而是有实际内容的作品**

- 先学最小的 Git 存档闭环
- 从访客视角决定补什么内容

---

## 第一步：Git 最小存档闭环（10 分钟）

### 为什么要现在学存档

- 改动开始变大（加内容模块、调页面结构）
- 一处改动可能影响多处
- 改崩了需要能退回来

### 基础版里的 Git，只负责三件事

| 动作   | 理解成什么                     |
| ------ | ------------------------------ |
| 存一次 | 给当前稳定版本打一个可回来的点 |
| 看历史 | 看看自己之前存过哪些版本       |
| 退回来 | 某次改崩时，回到上一个可用版本 |

### 让 AI 帮你做

**做存档**：

```text
我现在准备开始给主页补内容。

在动手前，请先帮我做一个最小存档，
让我之后改坏了还能退回来。

请一步一步带我完成，不要展开 Git 理论。
```

**查看历史**：

```text
请帮我查看这个项目目前有没有存档记录。

如果之后我改坏了，告诉我最小回退路径是什么。

只讲这次需要知道的部分。
```

### 为什么有了存档心态会不一样

- 更敢补内容
- 更敢试不同写法
- 更敢让 AI 做稍大一点的调整
- **即使改坏了，也不是世界末日**

## 第二步：从访客视角决定补什么（10 分钟）

### 先问一个关键问题

> 如果别人只花 30 秒看我的主页，他最应该记住什么？

这个问题会帮你从"我还能写什么"切换到"别人最该先看到什么"

### 三类访客通常关心什么

| 访客         | 最想先知道什么                           |
| ------------ | ---------------------------------------- |
| 朋友         | 你最近在做什么，有没有新作品或新方向     |
| HR / 面试官  | 你是谁，做过什么，能力大概在哪           |
| 潜在合作伙伴 | 你擅长什么，值不值得继续交流，怎么联系你 |

### 交集很稳定

你不需要给三类人做三套页面，只需要关注它们的交集：

- 你是谁
- 你做过什么
- 最近在做什么
- 怎么联系你

### 内容判断标准

这个模块加上去之后，能不能让访客更快回答这些问题？

- 这个人是谁？
- 他最近在做什么？
- 他做过哪些值得看的东西？
- 如果我想继续联系，应该怎么做？

**如果能明显帮助回答，就值得加；如果不能，就先别加**

## 第三步：添加 2-3 个最有价值的模块（30 分钟）

### 哪些模块最值得优先

| 模块          | 解决什么问题                   |
| ------------- | ------------------------------ |
| 作品展示区    | 让别人更快知道你做过什么       |
| 联系方式      | 让"感兴趣"可以变成"能联系到"   |
| 经历 / 时间线 | 帮别人快速看懂你的背景脉络     |
| FAQ           | 回答高频问题，降低访客开口成本 |

### 一次只加 2-3 个

**优先补最缺的**：

- 最缺"别人看不懂我做过什么" → 补作品展示区
- 最缺"别人想继续交流却不知道怎么找我" → 补联系方式
- 最缺"别人不知道该问什么" → 补 FAQ 或聊天区引导

### 让 AI 帮你加

```text
请帮我给个人主页补一个作品展示区和一个联系方式区。

目标是让访客更快知道我做过什么，以及怎么联系我。

请保持现在的整体风格，不要一次新增太多模块。
加完之后，再帮我检查聊天入口是否足够明显。
```

### 每加一类模块前先存档

**更稳的操作节奏**：

1. 存档
2. 加模块
3. 看效果
4. 再存档

这样你能清楚判断是哪一轮调整让页面更完整了

## 第四步：让数字分身更容易被使用（10 分钟）

### 不只是"有个聊天框"

很多页面的问题是：访客不知道该怎么用它

### 补一些引导

- 一两句更清楚的提示文案
- 几个快捷问题按钮
- 入口说明："你可以在这里问我最近在做什么、我有哪些作品、怎么联系我"

### 这类引导很重要

它会把"有一个聊天框"推进成"这个聊天框真的有人会用"
