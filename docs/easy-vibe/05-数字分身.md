# 第 5 章：让数字分身更像你

---

## 本章目标

**让数字分身不只是"会回话"，而是"会代表你说话"**

- 理解数字分身背后的链路
- 写第一版"数字分身说明书"
- 掌握四类常见问题的排查方法

---

## 第一步：理解"像你"比"更聪明"更重要（5 分钟）

### 评价标准

数字分身不是：

- ❌ 通用百科
- ❌ 全知全能客服

数字分身是：

- ✅ 代表你
- ✅ 回答和你有关的问题
- ✅ 不知道时诚实收住

### 先用一句人话理解背后发生了什么

```
访客提问
  ↓
项目把问题送出去
  ↓
模型结合说明书生成回答
  ↓
回答回到页面显示出来
```

### "数字分身说明书"是什么

一份告诉它以下内容的文件：

- 你是谁
- 你该怎么说
- 你知道什么
- 不知道什么时该怎么做

---

## 第二步：写第一版说明书（15 分钟）

### 说明书里最该写清什么

| 要写什么         | 为什么重要                   |
| ---------------- | ---------------------------- |
| 你是谁           | 决定它最基本的身份和事实边界 |
| 你知道什么       | 决定哪些问题它可以稳定回答   |
| 你怎么说话       | 决定回答的语气和表达风格     |
| 你不知道时怎么办 | 决定它会不会乱编             |

### 第一版骨架

```text
你是我的数字分身，用来在个人主页里回答访客关于我的问题。

你的任务：
- 介绍我是谁
- 回答和我有关的问题
- 帮访客了解我最近在做什么、做过什么、怎么联系我

关于我：
- 我是：______
- 我最近在做：______
- 我擅长或长期关注：______

说话方式：
- 语气：______
- 回答尽量：简洁 / 真诚 / 人话一点 / 不装专家

边界：
- 不要编造我没做过的经历
- 不要假装知道我没提供的信息
- 不知道时要明确说不知道，并建议访客通过联系方式进一步确认
```

**关键**：不用抽象性格词，用具体事实

---

## 第三步：用真实材料校准（10 分钟）

### 真正让它更像你的，是样本

不用很多，1-3 组真实材料就够了

### 示例

```text
示例 1
问：你现在主要在做什么？
答：我最近主要在整理自己的作品，
     也在尝试用 AI 做一些更完整的小项目。

示例 2
问：你擅长什么？
答：我比较擅长把复杂问题讲清楚，
     也比较关注 AI 应用、内容表达和知识整理这几个方向。
```

**重点不在多，而在真实**

---

## 第四步：四类常见问题排查（20 分钟）

### 先按症状分类

| 症状     | 最常见的方向               |
| -------- | -------------------------- |
| 不回复   | 链路、配置、接口没通       |
| 答偏     | 说明书不清、上下文不够     |
| 风格不对 | 抽象要求太多，真实样本太少 |
| 编造信息 | 不知道时的边界没写清       |

### 一次只改一个变量

**不回复**：

```text
现在的问题是：聊天不回复。

请先检查聊天请求有没有成功发出，
以及报错在哪一层。

先不要改前端代码，也不要换模型。
```

**答偏**：

```text
现在的问题是：聊天能回复，但回答经常答偏。

请先不要改前端代码，也不要换模型。
先帮我检查当前数字分身说明书缺了什么信息。
```

**风格不对**：

```text
现在它的事实基本对，但说话不像我。

请先基于我提供的 2 组真实表达样本调整语气，
不要同时改其他逻辑。
```

**编造信息**：

```text
现在它会编造我没做过的经历。

请强化"未知就收住"的规则，
不要补不存在的事实。
```

## 第五步：成本与安全意识（5 分钟）

### 成本：先建立意识

- 不要为了个人主页一开始就选过重、过贵的方案
- 回复能说清就行，不需要每次都写成长文

### 安全：守住底线

> **密钥不要直接写进代码仓库**
