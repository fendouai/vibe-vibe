# 第 0 章：开始前

---

## 00 这套教程在教什么

### 不是什么

- ❌ 不是先学语法、再学框架、再做练习的传统教材
- ❌ 不是要把你训练成专业工程师
- ❌ 不会把系统工程内容提前塞给你

### 真正要教什么

- ✅ **把想法说清楚** - 把模糊念头说成 AI 能执行的任务
- ✅ **学会和 AI 协作** - 你定义目标，AI 执行，你判断结果
- ✅ **先做出作品，再逐步理解** - 建立正反馈循环

## 03 卡住时怎么办

### 统一求助公式

```
我刚刚做了：[动作]

现在看到的问题是：[现象]

我原本希望得到的结果是：[目标]

请先帮我判断最可能的原因，再告诉我下一步该怎么做。
```

### 三个例子

**页面空白**

```
我刚刚让 AI 帮我修改了主页的布局。

现在页面打开后是空白的。

我本来想看到修改后的主页，请先帮我判断最可能的原因。
```

**按钮没反应**

```
我刚刚加了一个发送按钮。

现在点击之后没有任何反应。

我本来希望点击后能发送消息，请帮我定位最可能的问题。
```

**聊天结果不对**

```
我刚刚更新了数字分身的说明信息。

现在它回答得还是很机械。

我希望它更像我本人，请帮我判断是说明写得不够清楚。
```

### 迭代流程

```
发现问题 → 描述现象 → AI 给出判断 → 验证 → 解决了吗？
                                            ↓
                                       否 → 补充信息 → 继续
```

**记住**：

- 贴截图和完整报错
- 不要只说"不对"、"坏了"、"还是不行"
- 先看上下文、目标、边界，不要把责任全扔给 AI

## 04 学习路线图

```
第 1 章      第 2 章      第 3 章      第 4 章      第 5 章      第 6 章
先做 v1     带回本地     改更像作品   内容更完整   分身更像你   正式上线
  ↓           ↓           ↓           ↓           ↓           ↓
能预览       本地运行     风格统一     信息完整     回答自然     公网访问
```

### 推荐学习方式

- **边做边读，遇到问题再查** - 不是从头到尾连续阅读
- **每章最重要不是看完，而是交付** - 做出来，验收，再继续
- **附录是工具箱** - 遇到再查，不是预习任务

### 给自己留现实节奏

- 一次只完成一个小节
- 保证每次结束时，作品都比开始时更往前一点
- 比"今天看了很多"更重要："今天作品真的往前走了一步"
