## 开发历程

### Phase 1：初步探索（失败）

| 时间         | 事件                | 问题                     |
| ------------ | ------------------- | ------------------------ |
| 12.10 周三晚 | 开始用 Gemini3 开发 | 一步到位思维导致逻辑混乱 |
| 当晚         | 折腾到凌晨 4 点     | 结果越来越乱             |

**错误做法**：一次性实现所有复杂功能（贪吃蛇、俄罗斯方块思维）

### Phase 2：调整策略（成功）

| 时间           | 事件               | 收获         |
| -------------- | ------------------ | ------------ |
| 12.11 周四中午 | WhaleAI 群提问请教 | 得到指点     |
| 当晚           | 调整策略           | 效果立竿见影 |

**正确做法**：

| 要点     | 说明                         |
| -------- | ---------------------------- |
| 分步迭代 | 一次只聚焦一个模块           |
| 记录版本 | 方便回溯检查                 |
| 功能分开 | 基础布局 → 上传 → 提示词管理 |

---

## 核心教训：分步开发

| 错误方式                 | 正确方式             |
| ------------------------ | -------------------- |
| 一句话让 AI 实现所有功能 | 先写详尽开发文档     |
| 边想边写代码             | 先让 AI 输出技术方案 |
| 一步到位                 | 分版本逐步推进       |

**开发顺序**：

```
1. 基础界面布局
      ↓
2. 图片上传功能
      ↓
3. 提示词输入管理
      ↓
4. API 调用集成
      ↓
5. 逐步增加功能
```

## API 协议兼容问题

### 问题背景

| 平台   | 协议格式   | 问题                 |
| ------ | ---------- | -------------------- |
| OpenAI | 兼容性好   | 大多数 API 平台兼容  |
| Gemini | URL 写模型 | 官方和中转站格式不同 |

### 解决方案

| 步骤 | 操作                               |
| ---- | ---------------------------------- |
| 1    | 所有报错发给 API 平台技术支持      |
| 2    | 获取 Gemini 生图的 base64 格式示例 |
| 3    | 和 AI 聊几轮，搞兼容方案           |

## 设计理念

### 为什么纯前端 + HTML？

| 原因       | 说明                      |
| ---------- | ------------------------- |
| 硬件门槛低 | 学生无需升级电脑即可使用  |
| 即开即用   | 无需本地部署配置          |
| API 友好   | 充分利用已购买的 API 额度 |
| 部署简单   | 纯前端部署成本低          |

---

## 开发技巧总结

| 技巧     | 说明                 |
| -------- | -------------------- |
| 分步迭代 | 一次只做一件事       |
| 版本记录 | 每步保存，方便回溯   |
| 方案先行 | 先写开发文档再写代码 |
| 善用社区 | 遇到问题在群里请教   |
