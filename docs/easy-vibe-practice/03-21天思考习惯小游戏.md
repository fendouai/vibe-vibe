# 全栈开发：21天思考习惯小游戏

> **作者**：王星尧  
> **面向人群**：有一定 Vibe Coding 基础，想完成完整全栈项目的开发者  
> **项目周期**：约 2-3 周  
> **项目地址**：https://space-vqm.site/

---

## 本章目标

- 从 idea → PRD → UI → 开发 → 部署的全流程实践
- 掌握云开发环境（CNB）的使用
- 理解前后端协作流程
- 学会 EdgeOne/Vercel 部署方案

---

## 项目概览

| 维度         | 内容                                              |
| ------------ | ------------------------------------------------- |
| **创作初衷** | AI 时代下"遇问题就问 AI"导致主动思考能力退化      |
| **项目定位** | 21 天打卡挑战，唤醒主动思考习惯                   |
| **核心玩法** | 每日一个问题 → 填写思考 → 打卡成功 → 解锁记忆碎片 |

### 功能规则

| 功能     | 说明                         |
| -------- | ---------------------------- |
| 每日打卡 | 随机生成问题，填写思考即打卡 |
| 问题重置 | 不满意可强制重置一次         |
| 解锁奖励 | 完成打卡解锁机器人记忆碎片   |
| 注册登录 | 保存与修改答案记录           |

## 开发流程总览

```
阶段一：产品设计师
    ↓ Idea → MVP → PRD
阶段二：前端设计师
    ↓ Stitch → AI Studio
阶段三：后端架构师
    ↓ 云开发环境 → Claude Code
阶段四：部署上线
    ↓ EdgeOne / Vercel
```

---

## 阶段一：产品设计师

### Step 1：从想法到 MVP

**核心要点**：先明确目标，再与 AI 对话

| 步骤 | 操作                          |
| ---- | ----------------------------- |
| 1    | 明确产品目的和形式            |
| 2    | 把能想到的所有信息告诉 AI     |
| 3    | 多轮对话后让 AI 总结 MVP 版本 |

**对话技巧**：

| 技巧           | 说明                                      |
| -------------- | ----------------------------------------- |
| 让 AI 评估不足 | "这个想法有哪些不足，怎么解决？"          |
| 让 AI 检查漏洞 | "自己检查想法的逻辑漏洞"                  |
| 生成 10 个问题 | "生成 10 个产品问题，能回答证明 MVP 合格" |

### Step 2：从 MVP 到 PRD

**操作流程**：

1. 投入标准 PRD 模版 → https://www.vibevibe.cn/Advanced/03-prd-doc-driven/03-prd-template-guide.html
2. 使用 Gemini Canvas 边完善边修改
3. 导出 Markdown 格式 PRD

## 阶段二：前端设计师

### Step 1：Stitch 设计 UI

| 步骤 | 操作                            |
| ---- | ------------------------------- |
| 1    | PRD 投给 AI，生成 Stitch 提示词 |
| 2    | Stitch 生成初稿，对话修改       |
| 3    | 样式满意即可，文字用代码改      |

### Step 2：AI Studio 调整 UX

| 步骤 | 操作                          |
| ---- | ----------------------------- |
| 1    | 导入设计图 → Google AI Studio |
| 2    | 调整交互逻辑（跳转、按钮）    |
| 3    | Preview 实时预览效果          |
| 4    | 导入 GitHub 开始构建          |

---

## 阶段三：后端架构师

### Step 1：导入云开发环境

> 全线上开发，无需本地配置

| 步骤 | 操作                              |
| ---- | --------------------------------- |
| 1    | 打开 https://cnb.cool/ → 微信登录 |
| 2    | 新建仓库，填写信息                |
| 3    | 导入 GitHub 项目                  |

### Step 2：配置 .cnb.yml

```yaml
# .cnb.yml
$:
  vscode:
    - runner:
        cpus: 8
      docker:
        image: docker.cnb.cool/nfeyre/default-dev-env:latest
      services:
        - vscode
        - docker
      env:
        CNB_WELCOME_CMD: |
          npx @z_ai/coding-helper
      stages:
        - name: start 1P
          script: 1pctl start all
```

### Step 3：进入开发环境

| 步骤 | 操作                |
| ---- | ------------------- |
| 1    | 云原生构建          |
| 2    | 启动 Claude Code    |
| 3    | 配置 API Key（GLM） |

**API Key 获取**：https://bigmodel.cn/usercenter/proj-mgmt/apikeys

### Step 4：后端开发

| 步骤 | 操作                        |
| ---- | --------------------------- |
| 1    | 启动项目验证能否正常运行    |
| 2    | 输入 PRD 或添加文档到左侧栏 |
| 3    | AI 阅读项目 → 规划技术文档  |
| 4    | 分步执行，每步反馈修改      |

**常见问题**：AI Studio 导出的项目缺少入口文件

- 解决：拖入 `index.html` 和 `index.tsx`，告诉 AI 添加入口

### Step 5：提交

> 每天修改后必须提交保留记录

## 阶段四：部署上线

### EdgeOne 部署（国内访问）

| 步骤 | 操作                                                 |
| ---- | ---------------------------------------------------- |
| 1    | 打开 https://console.cloud.tencent.com/edgeone/pages |
| 2    | 导入 Git 仓库 → 选择 GitHub                          |
| 3    | 加速区域选"全球可用区（不含中国大陆）"               |

**构建设置**：

| 框架    | 输出目录 |
| ------- | -------- |
| Vite    | `dist`   |
| Next.js | `.next`  |

| 步骤 | 操作                      |
| ---- | ------------------------- |
| 4    | 配置环境变量（.env 内容） |
| 5    | 开始部署 → 获取预览地址   |
| 6    | 购买自定义域名            |
| 7    | 添加域名 → 验证归属权     |
| 8    | 添加 CNAME 记录           |
| 9    | 配置 HTTPS 证书           |

### Vercel 部署（境外访问）

> 免费域名，操作简单，但只能境外访问

| 步骤 | 操作                                |
| ---- | ----------------------------------- |
| 1    | 访问 vercel.com → GitHub 登录       |
| 2    | New Project → Import Git Repository |
| 3    | 配置环境变量                        |
| 4    | Deploy → 等待 1-2 分钟              |
| 5    | 验证部署结果                        |
