# AI 简历优化 SaaS

---

> **作者**：落尘
> **面向人群**：完成基础篇 + 进阶篇 Next.js 基础的开发者
> **项目周期**：约 1 周
> **最终成果**：一个通过 AI 优化简历、提高面试邀约率的 SaaS 产品

---

## 为什么选择这个项目

### 市场验证

2024 年简历构建工具市场规模 **16 亿美元**，预计 2032 年增长至 **31 亿美元**

### 成功案例

| 产品        | 成绩                          |
| ----------- | ----------------------------- |
| **Rezi.ai** | 330 万用户，年收入 240 万美元 |
| **Teal HQ** | 2070 万美元融资，200 万用户   |

**核心结论**：用户愿意为"提高求职成功率"付费

---

## 核心功能

| 功能         | 说明                                    |
| ------------ | --------------------------------------- |
| **简历诊断** | 上传简历，AI 指出排版、内容、关键词缺失 |
| **精准润色** | 根据 JD 对简历进行针对性修改            |
| **ATS 检测** | 确保简历能通过大公司筛选系统            |

## Phase 1：需求定义

### JTBD（Jobs to be Done）

| 表面任务     | 深层任务         |
| ------------ | ---------------- |
| 把简历改通顺 | **获得面试机会** |

### MVP

**只做三步**：

1. 左边框：粘贴简历文本（不做文件解析）
2. 右边框：粘贴职位描述（JD）
3. 中键：AI 生成优化建议

---

## Phase 2：Prompt 调试

### 错误 Prompt

```
帮我修改这份简历，让它更好。
```

（太模糊，AI 不知道什么是"好"）

### 优化后 Prompt

```markdown
# Role

你是资深技术招聘专家，精通 ATS 筛选规则。

# Context

简历内容：{RESUME_TEXT}
目标职位 JD：{JOB_DESCRIPTION}

# Task

分析简历与 JD 的匹配度，给出优化后的简历。

# Requirements

1. STAR 法则改写经历
2. 从 JD 提取关键词融入简历
3. 量化成果（"提升 50%"）
4. 专业、自信、客观语气

# Output Format

JSON 格式：{
"score": 0-100,
"missing_keywords": [...],
"analysis": "...",
"optimized_content": "..."
}
```

### Kimi K2 优势

**超长上下文**——可把公司产品文档、技术白皮书全部喂给它

## Phase 3：全栈开发

### 技术栈

| 技术          | 用途     |
| ------------- | -------- |
| Next.js 14    | 全栈框架 |
| Shadcn UI     | 组件库   |
| Tailwind CSS  | 样式     |
| Vercel AI SDK | 流式输出 |

### 复制这段话给 AI

```
我想做一个 AI 简历优化 SaaS。

技术栈：
- Next.js 14 (App Router)
- Shadcn UI + Tailwind CSS
- Vercel AI SDK

请帮我搭建项目基础结构：
1. 首页：Hero Section + "开始诊断"按钮
2. 工作台：左侧简历输入，右侧 JD 输入
3. 结果展示：Markdown 渲染的流式输出
```

### 接入 Kimi K2 API

```
Base URL: https://api.moonshot.cn/v1
Model: moonshot-v1-8k
API Key: 从环境变量读取
```

**使用 Vercel AI SDK 的 `streamText` 实现打字机效果**

## Phase 4：产品思考

### Aha Moment（惊喜时刻）

**先给一个评分**："你的简历当前评分：45 分。有 3 个致命关键词缺失..."

### 数据反馈闭环

记录：

- 用户最常复制的优化建议
- 用户在哪个步骤流失
- AI 生成是否截断

### 商业化

| 模式         | 说明                        |
| ------------ | --------------------------- |
| **Freemium** | 免费诊断，付费看优化建议    |
| **导出功能** | 精美 PDF 模板导出（付费点） |

---

## 踩坑指南

| 坑           | 解决方案                        |
| ------------ | ------------------------------- |
| **隐私问题** | 要求 AI 脱敏敏感信息            |
| **幻觉问题** | Prompt 加约束：严禁编造经历     |
| **排版乱掉** | 使用 `react-markdown` + `prose` |
| **部署报错** | Vercel 控制台配置环境变量       |

---

## 总结

**最重要的不是"怎么调用 API"，而是**：

1. **Mindset**：敢于做减法，只验证文本优化
2. **Prompt**：像招聘专家思考，不是像程序员思考
3. **Vibe**：流式输出缓解等待焦虑
