## 快速开始（5 分钟）

### Windows 用户

#### 1. 安装 Git

**下载**：https://npmmirror.com/mirrors/git-for-windows/v2.52.0.windows.1/Git-2.52.0-64-bit.exe

双击安装，一直点"下一步"。

#### 2. 安装 Node.js

**下载**：https://npmmirror.com/mirrors/node/v24.13.0/node-v24.13.0-x64.msi

双击安装，一直点"下一步"。

#### 3. 验证安装

按 `Win + X`，选择**终端（管理员）**，执行：

```powershell
git --version
node -v
```

#### 4. 配置镜像并安装 pnpm

```powershell
npm config set registry https://registry.npmmirror.com/
npm install -g pnpm
pnpm setup
pnpm config set registry https://registry.npmmirror.com/
```

#### 5. 安装 Claude Code

```powershell
npm install -g @anthropic-ai/claude-code
claude
```

### Mac/Linux 用户

#### 1. 安装 Git

**macOS**：首次在终端输入 `git` 会提示安装，或执行 `xcode-select --install`

**Linux**：`sudo apt install git -y`

#### 2. 执行初始化脚本

**macOS**：

```bash
curl -fsSL https://cnb.cool/vibevibe/scripts/-/git/raw/main/init-node-mac.sh | sed 's/\r$//' | sh
```

**Linux**：

```bash
curl -fsSL https://cnb.cool/vibevibe/scripts/-/git/raw/main/init-node-linux.sh | sed 's/\r$//' | sh
```

#### 3. 验证安装

```bash
git --version
node -v
```

#### 4. 安装 Claude Code

```bash
npm install -g @anthropic-ai/claude-code
claude
```

## 代码格式演变

### 网页的三层结构

| 层次                 | 作用           | 示例                 |
| -------------------- | -------------- | -------------------- |
| HTML（结构层）       | 页面有什么内容 | `<div>`, `<button>`  |
| CSS（样式层）        | 长什么样       | `color`, `font-size` |
| JavaScript（行为层） | 怎么交互       | `addEventListener`   |

### 代码格式的四个阶段

```
单文件 HTML → 文件分离 → 模块化 → TypeScript 工程化
```

| 阶段 | 格式                    | 适用场景    | 运行方式        |
| ---- | ----------------------- | ----------- | --------------- |
| 1    | 单文件 HTML             | 简单 Demo   | 直接打开        |
| 2    | HTML+CSS+JS 分离        | 超过 200 行 | 需要服务器      |
| 3    | 模块化（import/export） | 多人协作    | 需要 build      |
| 4    | TypeScript + 构建工具   | 复杂项目    | 需要 dev server |

### TypeScript 特点

- **不能直接在浏览器运行**，需要编译
- 开发时：`pnpm dev` 自动编译
- 上线时：`pnpm build` 打包优化
- AI 喜欢用 TypeScript：类型安全，减少错误

## 技术栈分层

现代 Web 应用分三层：

```
用户界面 → 前端层 → 后端层 → 数据层
```

| 层级   | 技术         | 作用                 |
| ------ | ------------ | -------------------- |
| 框架   | Next.js      | 前后端一体           |
| 语言   | TypeScript   | 类型安全             |
| 样式   | Tailwind CSS | 实用优先的 CSS       |
| 组件库 | shadcn/ui    | 可复用 UI 组件       |
| 数据库 | PostgreSQL   | 关系型数据库         |
| ORM    | Drizzle ORM  | 类型安全的数据库操作 |

### 为什么选 Next.js

- **AI 理解成本低**：前后端在一个项目，结构统一
- **部署零成本**：Vercel/EdgeOne 免费托管
- **npm 生态**：200 万+ 开源包可用

### 常用"轮子"

| 功能        | 推荐包                  |
| ----------- | ----------------------- |
| 表单验证    | `zod`                   |
| 表单管理    | `react-hook-form`       |
| 数据请求    | `@tanstack/react-query` |
| 时间处理    | `dayjs`                 |
| HTTP 客户端 | `axios`                 |
| 图标        | `lucide-react`          |

## 浏览器与服务器

|          | 浏览器（客户端）      | 服务器（服务端）     |
| -------- | --------------------- | -------------------- |
| 职责     | 渲染页面、执行交互    | 处理业务、查询数据库 |
| 能运行   | HTML、CSS、JavaScript | Node.js、Python、Go  |
| 不能运行 | TypeScript、后端语言  | 浏览器 API           |

### 为什么需要 Node.js

- 运行 TypeScript 编译器
- 运行构建工具（Vite、Webpack）
- 启动开发服务器

### 开发环境 vs 生产环境

|      | 开发环境（Localhost） | 生产环境（公网）      |
| ---- | --------------------- | --------------------- |
| 位置 | 你的电脑              | 远程服务器            |
| 地址 | `localhost:3000`      | `https://example.com` |
| 代码 | 未压缩、带调试信息    | 压缩、混淆            |
| 更新 | 热重载                | 需要重新部署          |

## Terminal 终端入门

### 打开终端

- **Mac**：`Cmd + Space`，输入 "Terminal"
- **Windows**：`Win + R`，输入 "powershell"

### 提示符

```bash
user@MacBook ~ $     # Mac 提示符是 $
PS C:\Users\user>    # Windows 提示符是 >
```

**提示符不是命令的一部分**，输入命令时不要复制它。

### 常用命令

| 命令             | 作用                             |
| ---------------- | -------------------------------- |
| `ls`             | 列出文件（Mac/Linux/PowerShell） |
| `cd <目录>`      | 切换目录                         |
| `pwd`            | 显示当前路径                     |
| `mkdir <目录名>` | 创建文件夹                       |
| `rm <文件>`      | 删除文件                         |
| `cp <源> <目标>` | 复制文件                         |
| `mv <源> <目标>` | 移动/重命名文件                  |

### 快捷键

| 快捷键   | 作用         |
| -------- | ------------ |
| `Tab`    | 自动补全     |
| `↑↓`     | 浏览历史命令 |
| `Ctrl+L` | 清屏         |
| `Ctrl+C` | 取消当前命令 |

### 报错排查

看到 `command not found` 时：

1. 检查命令拼写
2. 确认工具是否已安装
3. 检查当前目录是否正确

## 包管理

### nvm（Node 版本管理器）

- 在同一台电脑上管理多个 Node.js 版本
- 不同项目可以使用不同版本

```bash
nvm install 24.13.0  # 安装指定版本
nvm use 24.13.0      # 使用指定版本
nvm list             # 查看已安装版本
```

### pnpm vs npm

| 特性               | npm  | pnpm      |
| ------------------ | ---- | --------- |
| 速度               | 基准 | 快 2-3 倍 |
| 磁盘空间（10项目） | ~5GB | ~1.5GB    |

### pnpm 常用命令

| 命令                 | 作用           |
| -------------------- | -------------- |
| `pnpm init`          | 初始化项目     |
| `pnpm install`       | 安装所有依赖   |
| `pnpm add <包名>`    | 安装生产依赖   |
| `pnpm add -D <包名>` | 安装开发依赖   |
| `pnpm dev`           | 运行开发服务器 |

### 全局安装 vs 项目安装

|      | 全局安装             | 项目安装              |
| ---- | -------------------- | --------------------- |
| 命令 | `npm install -g xxx` | `pnpm add xxx`        |
| 位置 | 系统目录             | 当前项目 node_modules |
| 用途 | CLI 工具             | 项目依赖              |
| 示例 | claude、http-server  | react、lodash         |

### 核心配置文件

| 文件             | 作用                 |
| ---------------- | -------------------- |
| `package.json`   | 项目元数据和依赖声明 |
| `pnpm-lock.yaml` | 锁定依赖版本         |
| `tsconfig.json`  | TypeScript 配置      |

## Localhost 与端口

### Localhost

| 地址                    | 作用                       |
| ----------------------- | -------------------------- |
| `localhost`             | 本地主机的别名             |
| `127.0.0.1`             | 本地回环 IP 地址           |
| `http://localhost:3000` | 本地运行的第 3000 端口服务 |

### 端口

如果把 IP 地址比作**大楼地址**，端口就是**房间号**。

| 端口      | 服务             |
| --------- | ---------------- |
| 3000      | Next.js、Express |
| 5173      | Vite             |
| 8000/8080 | 通用备用端口     |

**规则**：

- **0-1023**：系统端口，需要管理员权限
- **1024-49151**：注册端口，常见服务
- **49152-65535**：动态端口，临时使用

**一个端口同时只能运行一个服务**

看到 `EADDRINUSE` 错误说明端口被占用，换一个端口即可。

### 启动开发服务器

```bash
cd my-app          # 进入项目目录
pnpm install       # 安装依赖（首次）
pnpm dev           # 启动开发服务器
```

访问：`http://localhost:3000`

## 常见问题

### Q：为什么要配置国内镜像源？

解决下载慢的问题，使用淘宝镜像加速。

### Q：必须用 GLM 吗？

不是。可以使用其他国内模型或 Claude。

### Q：TypeScript 和 JavaScript 有什么区别？

TypeScript 增加了类型检查，开发阶段就能发现错误。

### Q：为什么有些网页保存后打不开？

现代网站使用 React/Next.js 等框架，代码分离，需要服务器才能运行。

### Q：安装过程中遇到错误怎么办？

把错误信息复制给 AI，它会帮你解决。
