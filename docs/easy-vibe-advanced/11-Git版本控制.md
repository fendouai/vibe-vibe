## "压缩包协作"困境

### 场景

小明把项目 zip 发给朋友，朋友改了发回来 → 两个人的版本不一样，不知道用谁的

### Git 解决的问题

- 代码版本管理
- 多人协作不打架
- 追踪每次改动

---

## Git 三区模型

| 区域       | 内容             |
| ---------- | ---------------- |
| **工作区** | 当前修改的文件   |
| **暂存区** | 准备提交的文件   |
| **仓库区** | 已提交的历史记录 |

### 工作流

```
修改文件 → git add → git commit → git push
```

---

## 常用命令

| 命令                      | 作用                 |
| ------------------------- | -------------------- |
| `git add .`               | 添加所有修改到暂存区 |
| `git commit -m "message"` | 提交并写说明         |
| `git push`                | 推送到远程仓库       |
| `git pull`                | 拉取最新代码         |
| `git log`                 | 查看提交历史         |
| `git diff`                | 查看修改内容         |

## 推送到云端

### GitHub

- 创建仓库
- 推送代码
- 朋友可以 clone 下载

### SSH 认证

配置 SSH Key 后，推送不需要每次输入密码

---

## 冲突解决

### 什么是冲突

两个人同时修改同一个文件，Git 不知道用谁的版本

### 解决方法

1. Git 会标记冲突位置
2. 手动选择保留哪个版本
3. 提交解决后的代码

---

## .gitignore

### 作用

告诉 Git 哪些文件不要追踪

### 常见配置

```
.env           # 环境变量（包含密钥）
node_modules/  # 依赖文件
.DS_Store      # Mac 系统文件
```

## 分支与 PR

### 分支

**分支 = 独立的开发线**

- 主分支（main）：稳定版本
- 开发分支：新功能开发

### Pull Request

**PR = 请求合并分支**

- 提交前先创建分支
- 开发完成后提交 PR
- 团队审核后合并
