## Serverless 部署

### 什么是 Serverless

**不是没有服务器，是你不需要管理服务器**

- 只管写代码
- 平台负责扩容、维护

### 推荐平台

| 平台                 | 特点                               |
| -------------------- | ---------------------------------- |
| **EdgeOne Pages**    | 国内可访问，免费，Next.js 支持好   |
| **Vercel**           | Next.js 官方平台，体验极佳，国内慢 |
| **Cloudflare Pages** | 全球 CDN，免费额度慷慨             |

## CI/CD 自动化

### 传统部署

手动打包 → 传到服务器 → SSH 登录 → 重启服务

### CI/CD 部署

```
git push → GitHub 自动通知 → 平台自动构建 → 自动发布
```

**保存即发布**

---

## 部署配置

### 构建设置

| 配置项    | 说明         |
| --------- | ------------ |
| 框架预设  | 选择 Next.js |
| 根目录    | 默认 `./`    |
| 构建命令  | `pnpm build` |
| 输出目录  | `.next`      |
| Node 版本 | 24           |

### 环境变量

**部署后需要单独配置**

- GitHub 上没有 `.env` 文件
- 在平台的环境变量页面配置
- 国内平台支持批量导入

---

## 部署踩坑

### 常见问题

| 问题          | 原因               | 解决           |
| ------------- | ------------------ | -------------- |
| 构建失败      | Node 版本不一致    | 调整 Node 版本 |
| 页面报错      | 环境变量缺失       | 配置环境变量   |
| 403 Forbidden | 默认链接有访问限制 | 绑定域名       |

## 成本优化

### 分阶段考虑

| 阶段   | 用户数     | 方案              |
| ------ | ---------- | ----------------- |
| MVP    | <1000      | Serverless 零成本 |
| 增长期 | 1000-10000 | 混合架构          |
| 企业期 | >10000     | 自建服务器        |

**过早优化是万恶之源**

### AI 模型成本

- 简单任务用国产模型（GLM、DeepSeek）
- 复杂推理用 Claude
- 智能路由可降低 80% 成本
