## CSS 与组件

| -------- | --------------------------- |
| **CSS**  | 网页的"化妆品"——决定样式    |
| **组件** | 预制的"乐高积木"——可复用    |
| **布局** | 元素摆放方式——Flexbox、Grid |

---

## Tailwind CSS

### 核心优势

传统 CSS 写法像**写作文**（容易撞名），Tailwind 像做**填空题**

```html
<!-- Tailwind：直接在标签里写样式 -->
<button class="p-4 bg-red-500 rounded-lg">提交</button>
```

**AI 最擅长做填空题**

---

## shadcn/ui

### 与传统组件库的区别

| 传统组件库                    | shadcn/ui                    |
| ----------------------------- | ---------------------------- |
| 安装在 node_modules（黑盒子） | 源代码复制到项目（完全控制） |
| 无法修改内部结构              | AI 可直接修改积木内部        |

**shadcn/ui 是为 AI 编程量身定做的模式**

### 推荐搭配

- shadcn/ui（组件库）
- Lucide React（图标库）——避免 AI 瞎编图标名

## UX 避坑指南

### 按钮 Loading 状态

**错误示范**：点击"支付"后无反应，用户疯狂点击

**正确做法**：

```
点击后按钮变灰 + 显示转圈动画 + 禁止再次点击
```

### 破坏性操作防护

**错误示范**：点击删除，数据瞬间消失

**正确做法**：

```
AlertDialog 警告弹窗 + 要求确认 + 甚至要求输入 "DELETE"
```

### 空状态处理

**错误示范**：新用户进来，页面一片空白

**正确做法**：

```
可爱插画 + "创建第一个项目"引导按钮
```

## 响应式设计

### 核心概念

**同一套代码，根据屏幕尺寸自动调整布局**

| 设备 | 布局 |
| ---- | ---- |
| 手机 | 单列 |
| 平板 | 双列 |
| 电脑 | 多列 |

### 提示词

```
适配移动端
手机上显示单列
```

## 从 UI 到 UX 的跃迁

### UI vs UX

|                | 定义           |
| -------------- | -------------- |
| **UI（界面）** | 表象——好不好看 |
| **UX（体验）** | 本质——好不好用 |

### VibeCoding 的产品思维

| 你负责                       | AI 负责                        |
| ---------------------------- | ------------------------------ |
| 理解用户是谁、在什么场景使用 | 根据场景实现细节               |
| 定义场景："用户此时很紧张"   | 加 Loading、本地保存、明确提示 |

**好设计不是技术炫技，而是理解用户**

## 性能意识

### 图片控制

- 高清大图可能几 MB
- 移动网络用户要等十几秒
- 控制图片大小，必要时让 AI 帮你压缩

### 提醒

性能优化不需要现在深究，上线后平台会自动做很多优化
